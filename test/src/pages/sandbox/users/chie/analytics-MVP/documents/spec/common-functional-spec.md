
# Common Functional Spec

このドキュメントは、analytics-MVPの全画面・全タブ・全カードに共通するUI仕様・集計仕様・状態管理・権限・グラフ・テーブル等の横断的なルールをまとめたものです。


## 画面・UI共通仕様

- `UI-COMMON-001` 画面タイトルは `レポート` とする。関連ID: `BH-001`
- `UI-COMMON-002` 画面トップタブは `案件数 / 所要時間 / 現在の業務状況` とする。関連ID: `BH-001`
- `UI-COMMON-009` 集計期間入力は開始/終了を年Select+月Selectで指定し、表示は `ja-JP: YYYY年 + M月`、`en-US: Year + short month` とする。関連ID: `BH-001`
- `UI-COMMON-010` ユーザー選択はTagPickerで行い、ユーザー/ユーザーグループの両方を選択可能とする。関連ID: `BH-001`
- `UI-COMMON-011` ユーザー未選択時は全ユーザーを対象とし、ユーザーグループ選択時は所属ユーザー全員を対象とする。関連ID: `BH-001`
- `UI-COMMON-012` 集計期間の初期値は直近6ヶ月（今日から6ヶ月前の月初〜今月末）とする。関連ID: `BH-001`
- `UI-COMMON-014` テーブルUIは、列表示設定・固定列などの表示設定を扱え、別ページへ遷移して戻った後も設定を復元する。関連ID: `BH-005, BH-006`
- `UI-COMMON-016` データ0件・取得失敗などの状態では、通常表示と区別できるフィードバックを出し、再操作が可能な導線を提供する。関連ID: `BH-001`

### カード共通UI

- `UI-COMMON-003` 各カードは、同一のヘッダー構成（タイトル、任意の情報アイコン、任意の操作群）で表示する。関連ID: `BH-001`
- `UI-COMMON-004` フィルターボタンは、クリックでPopoverを開いて条件選択を行う。関連ID: `BH-001`
- `UI-COMMON-005` フィルターがアクティブな場合はバッジを表示し、リセットボタンで全条件を初期値へ戻せる。関連ID: `BH-001`
- `UI-COMMON-006` 表示形式切替は `グラフ / テーブル` を選択でき、初期表示はグラフとする。関連ID: `BH-001`
- `UI-COMMON-007` 表示軸切替はSelectコンポーネントで行い、現在値を明示する。関連ID: `BH-001`
- `UI-COMMON-008` 情報アイコンはカードタイトル横に表示し、ホバーで説明Popoverを表示、クリックでピン留めを切り替える。関連ID: `BH-001`
- `UI-COMMON-013` Checkbox（例: 昨年を表示）はON/OFFを即時反映する。関連ID: `BH-001`
- `UI-COMMON-017` `昨年を表示` がONの間は、今年データに加えて昨年同期データを同時に表示し、ON/OFFで表示内容を即時切り替える。関連ID: `BH-001`


### Pane連携UI

- `UI-COMMON-015` ユーザーや案件のドリルダウン操作は、遷移前のフィルター条件・表示切替・表示設定を維持できる。関連ID: `BH-005, BH-006`


## 集計対象の共通仕様

- `ASSIGNEE-COMMON-001` 集計対象オプションは `主担当案件 / 副担当案件 / 主担当 + 副担当` とし、初期値は `主担当案件` とする。関連ID: `BH-001`
- `ASSIGNEE-COMMON-002` `主担当 + 副担当` 表示時は、主担当と副担当を別系列として集計し、重複カウントを許容する。関連ID: `BH-001`
- `ASSIGNEE-COMMON-003` 主担当は `assignee`、副担当は `subAssignee` を集計キーとする。関連ID: `BH-001`
- `ASSIGNEE-COMMON-004` ユーザー詳細Paneでは、集計対象設定を引き継がず、主担当/副担当の両方を表示する。関連ID: `BH-001`


## 画面状態の保存ルール

- `STATE-COMMON-001` テーブル表示設定（カラム移動/表示非表示/幅調整/固定）とソート設定は、案件詳細ページへ遷移して戻った後も復元する。関連ID: `BH-006`
- `STATE-COMMON-002` フィルター条件は、案件詳細ページへ遷移して戻った後も復元する。関連ID: `BH-006`
- `STATE-COMMON-003` 表示軸・表示形式の設定は、案件詳細ページへ遷移して戻った後も復元する。関連ID: `BH-006`


## 権限・認可ルール

- `AUTH-COMMON-001` Proライセンス保有ユーザーは、グローバルメニューで「レポート」を表示でき、レポートページを閲覧できる。関連ID: `US-001, BH-001`
- `AUTH-COMMON-002` Proライセンス非保有ユーザーは、グローバルメニューで「レポート」を表示できず、レポートページを閲覧できない。関連ID: `US-001, BH-001`
- `AUTH-COMMON-003` 集計範囲に閲覧可能/不可の案件が混在する場合は、閲覧可能な案件のみを表示・集計対象にする。関連ID: `US-009, BH-001`
- `AUTH-COMMON-004` 集計範囲に削除済み案件が含まれる場合は、削除済み案件を表示・集計対象外にする。関連ID: `US-010, BH-001`


## グラフ表示の共通ルール

- `GRAPH-COMMON-001` 高さはデータ件数に応じて動的計算し、最小240px・最大640pxを基準にする。関連ID: `BH-001`
- `GRAPH-COMMON-002` 高さが最大値を超える場合は縦スクロール（`overflowY: auto`）を有効化する。関連ID: `BH-001`
- `GRAPH-COMMON-003` X軸は必要に応じて非表示、Y軸はカテゴリ（ユーザー名など）表示を基本とする。関連ID: `BH-001`
- `GRAPH-COMMON-004` 目盛幅は表示ラベル領域を加味した余裕を持たせ、集計対象切替時に再計算する。関連ID: `BH-001`
- `GRAPH-COMMON-005` 集計対象が `主担当 + 副担当` または `副担当案件` の場合は、Y軸ラベルで `主/副` を表示する。関連ID: `BH-001`
- `GRAPH-COMMON-006` 月ラベルクリック時は、該当条件で案件一覧を別タブで開く。関連ID: `BH-001`
- `GRAPH-COMMON-007` `昨年を表示` がONの場合、グラフの系列・凡例・ツールチップは `今年` と `昨年` を識別できる表示に切り替える。関連ID: `BH-001`


## テーブルビューの共通ルール

- `TABLE-COMMON-001` 数値セルは整数または小数第1位まで表示し、日数項目には `日` を付与する。関連ID: `BH-001`
- `TABLE-COMMON-002` 集計対象が `主担当案件` の場合、担当列ヘッダーは `（主）` を付けず、`副` 系列ヘッダーは表示しない。関連ID: `BH-001`
- `TABLE-COMMON-003` 集計対象が `副担当案件` の場合、担当列ヘッダーは `（副）` 付きで表示し、`主` 系列ヘッダーは表示しない。関連ID: `BH-001`
- `TABLE-COMMON-004` 集計対象が `主担当 + 副担当` の場合、担当列ヘッダーは `（主）` と `（副）` を併記し、同一指標内の列順は常に `主 → 副` とする。関連ID: `BH-001`
- `TABLE-COMMON-005` テーブル列の基本順は `ユーザー名 → 合計 → 指標内訳` とし、`主担当 + 副担当` のときは `合計（主）→ 合計（副）`、内訳も各指標で `主 → 副` の順にする。関連ID: `BH-001`
- `TABLE-COMMON-006` 各行の `開く` 操作は、現在のフィルター条件と集計期間を維持してPaneを開く。関連ID: `BH-001, BH-006`
- `TABLE-COMMON-008` 月次テーブルの月並びは時系列昇順（古い月 → 新しい月）を基本とする。関連ID: `BH-001`
- `TABLE-COMMON-009` `昨年を表示` がONの場合、テーブル列ヘッダーは `今年` / `昨年` を識別できる表記に切り替える。関連ID: `BH-001`
- `TABLE-COMMON-010` `昨年を表示` がONの場合、テーブル列は各指標で `今年 → 昨年` の順に表示する。関連ID: `BH-001`

## ソート・表示順の共通ルール

- `SORT-COMMON-001` ソートキーは案件数またはユーザー名を選択でき、グラフ/テーブルで共通の並び順を維持する。関連ID: `BH-001, BH-006`

## ユーザー詳細Paneへの遷移

- `PANE-NAV-COMMON-001` グラフ要素クリック時は、クリック要素に対応するフィルターをセットしてPaneを開く。関連ID: `BH-001`
- `PANE-NAV-COMMON-002` クリックしたフィルター以外の条件は、遷移元の設定を維持して引き継ぐ。関連ID: `BH-001, BH-006`
- `PANE-NAV-COMMON-003` 業務負荷/時間タブからPane遷移する場合は、集計期間を引き継ぐ。関連ID: `BH-001`
- `PANE-NAV-COMMON-004` テーブルの `開く` 操作でも、遷移元のフィルター条件と集計期間を引き継ぐ。関連ID: `BH-001, BH-006`
- `PANE-NAV-COMMON-005` Pane初期タブは遷移元コンテキストに応じて決定する（現在の業務状況: 進行中、業務負荷/時間: パフォーマンス）。関連ID: `BH-001`

## API仕様（リクエスト/レスポンス差分など）

- `API-001` 案件データ取得APIは、フィルター条件と集計対象設定をパラメータとして受け取り、`HTTP 200` を成功、`HTTP 4xx` をエラーとして扱う。関連ID: `BH-001`
- `API-002` ユーザー/ユーザーグループ取得APIは、ユーザー情報とユーザーグループ情報を取得し、`HTTP 200` を成功、`HTTP 4xx` をエラーとして扱う。関連ID: `BH-001`

## データ仕様・永続化

- `DATA-001` 画面状態データは、ユーザー単位でローカルストレージに JSON 形式で永続化する。関連ID: `BH-005`
- `DATA-002` 案件データは、閲覧可能な案件のみを対象として削除済み案件を除外する。関連ID: `BH-001`
- `DATA-003` ステータス履歴データは、`CaseStatusHistory` からステータス変更履歴と完了日時を取得する。関連ID: `BH-002`
- `DATA-004` メッセージデータは、`CaseMessage` から初回返信日時を取得する。関連ID: `BH-003`

## バリデーション・エラー仕様

- `ERR-001` 集計期間が選択範囲外の場合は、フォームに「集計期間は過去24ヶ月以内で選択してください」を表示する。関連ID: `BH-001`
- `ERR-002` データ取得に失敗した場合は、画面に「データの取得に失敗しました。しばらく時間をおいて再度お試しください」を表示する。関連ID: `BH-001`

## 非機能要件（品質特性）

- `NFR-001` 画面表示のレスポンスは 2 秒以内とし、画面読み込みからデータ表示までの時間で測定する。関連ID: `US-001`
- `NFR-002` ローカルストレージの容量は 10MB 以内とし、保存データのサイズ監視で検証する。関連ID: `BH-005`

## 仕様ID命名ルール

- 形式: `SPEC-REPORT-<SCOPE>-<TOPIC>-<NNN>`
- `SCOPE` 例: `COMMON`, `TAB`, `PANE`
- `TOPIC` 例: `STATE`, `AUTH`, `FILTER`, `CHART`
- `NNN`: 3桁連番（同一 `SCOPE` + `TOPIC` 内で採番）
