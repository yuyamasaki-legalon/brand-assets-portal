# analytics-MVP/documents/spec 修正方針（ドラフト）

## 目的

- analytics-MVP配下documents/specディレクトリの仕様書を、AI実装・自動生成に適した形式へ正規化する

## 方針（抽象化）

1. **スコープ**: AI向け仕様書のみ対象、`reference`/`archive`/`source`等は除外
2. **命名・用語**: プロトタイプ固有名・旧プロジェクト名を排除し、機能的な名称に統一
3. **参照記法**: セクション番号参照（例: `3.4`）は `ファイル+セクション` 明記に置換
4. **構造**: 目的・用語・レイアウト・共通仕様・共通スタイルを分離
5. **重複排除**: 共通ルールは一箇所に集約し、重複を排除
6. **可搬性**: ローカルパス/個人ID/プロトタイプ実装依存の記述を除去
7. **記述形式**: 箇条書き中心、1要件1行、書き振りを統一
8. **最終チェック**: 文字化け、旧用語、不要参照、重複セクションを検出して修正

## このプロジェクト固有ルール（analytics-MVP）

1. **名称統一**:
   - `チームユーザー分析` は `レポート機能` に統一
   - `（analytics-MVP team-members）` のような旧識別子は削除
2. **参照記法**:
   - `3.4参照` のような番号参照は禁止
   - 必ず `ファイル名 + セクション名` で参照
3. **記述形式**:
   - 文章より箇条書きを優先
   - 1ファイル内で書き振りを統一
   - 1文1行で改行を維持
   - セクション多めの型を優先し、短い内容でも見出しで区切る
   - カード仕様の推奨順序は `カード概要 → 表示・操作 → グラフ表示（該当時） → テーブル表示（該当時） → 集計・データルール → 画面状態の保存（該当時） → 検証観点` とする
4. **重複排除**:
   - `機能仕様` と `シナリオ仕様` の重複は統合し、原則 `機能仕様` に一本化
   - 内容がない `スタイリング仕様` セクションは削除
5. **タブ/カードの責務分離**:
   - `tab-spec.md` はタブ共通事項のみ記載
   - カード固有事項は `card-*.md` 側にのみ記載
6. **共通仕様の集約**:
   - カード個別に依存しない仕様は `common-functional-spec.md` に集約
   - `common-functional-spec.md` にはカード固有仕様を置かない
7. **チャート仕様の運用**:
   - 共通チャート仕様は `common-chart-style-and-color-spec.md` に集約
   - `参照元コード一覧` は記載しない（プロトタイプ依存情報のため）
8. **トークン・値の記載**:
   - `blue-600` のような数値階調ラベルは禁止
   - `base` のようなプロトタイプ由来ラベルは禁止
   - Aegisトークン名を優先し、必要な場合のみ数値を併記

## 運用

- この方針に従い、documents/spec配下の仕様書を修正・管理する
