# Turnaround - Card 1: 案件の対応時間

## 機能仕様
### カード概要

- 目的: チーム全体の月次リードタイム・初回返信速度の推移を把握する
- カード名: 案件の対応時間

### 表示・操作

- 情報アイコン: `common-functional-spec.md`「画面・UI共通仕様 / カード共通UI」参照
- 集計期間表示: `common-functional-spec.md`「画面・UI共通仕様 / カード共通UI」参照
- 表示形式: `グラフ / テーブル`
- 初期表示: `グラフ`
- 昨年表示トグル: ON/OFF で今年と昨年の比較表示を切り替える
- フィルター: 案件タイプ
- フィルター反映: 本カードの表示・集計に即時反映する
- フィルターリセット: 案件タイプが「すべて」以外のときのみ表示し、クリックで案件タイプを「すべて」に戻す

### グラフ表示

- グラフ種別: 複合線グラフ
- X軸: 月
- Y軸: 日数
- 系列: 完了までの時間（中央値） / 初回返信（中央値）
- 昨年表示ON時: 昨年系列を追加表示し、今年系列と識別可能にする
- 凡例: 各系列名を表示する（昨年系列は「(昨年)」付き）
- 全期間サマリー: グラフ表示時のみ、グラフ上部に全期間の中央値を表示する
- 全期間サマリーの表示値: 値が 0 以下の場合は `-` を表示する
- 全期間サマリーの昨年値: 昨年表示ONかつ昨年値がある場合のみ表示する

### テーブル表示

- カラム構成:
  - `月`（固定カラム）
  - `完了までの時間`（または `完了までの時間 (今年)`）
  - `完了までの時間 (昨年)`（昨年表示ON時のみ）
  - `初回返信`（または `初回返信 (今年)`）
  - `初回返信 (昨年)`（昨年表示ON時のみ）
- 行構成:
  - 1行目: 全期間（選択期間全体の中央値）
  - 2行目以降: 月別データ
- 月別行の並び順: 時系列の昇順（古い月 → 新しい月）
- 値表示: 小数第1位 + `日`
- 欠損/0値表示: `-`

### 集計・データルール

- リードタイム/初回返信の定義: `terms-and-definitions.md` 参照
- 月次集計の定義: `terms-and-definitions.md` 参照
- 上部入力（ユーザー・ユーザーグループ/集計期間）の定義: `Turnaround/tab-spec.md` 参照
- 昨年比較期間: 選択期間を 1 年前にシフトした同期間
- 昨年データ欠損時: 補完値は生成せず、欠損値として扱う（表示は `-`）

### 検証観点

- `TURNAROUND-LEADTIME-VAL-001` 初期表示はグラフで、2指標（完了までの時間/初回返信）が表示される
- `TURNAROUND-LEADTIME-VAL-002` 昨年表示ON時、グラフとテーブルの両方で今年/昨年を識別できる
- `TURNAROUND-LEADTIME-VAL-003` 完了案件がない月は、該当指標が `-` 表示となる
- `TURNAROUND-LEADTIME-VAL-004` テーブル表示時、1行目が全期間、2行目以降が月昇順で表示される
- `TURNAROUND-LEADTIME-VAL-005` 案件タイプフィルター変更時、本カードの集計結果が更新される
- `TURNAROUND-LEADTIME-VAL-006` フィルターリセット時、案件タイプが「すべて」に戻る
