# Turnaround - Card 2: 案件の完了までの時間の内訳

## 機能仕様
### カード概要

- 目的: ユーザー別に、完了までの時間の内訳（ステータス分類ごとの日数）を把握する
- カード名: 案件の完了までの時間の内訳
- 対象外: 完了案件数の概要表示は `Volume/card-member-overview.md` を参照
- 対象外: 初回返信速度の概要表示は `Turnaround/card-lead-time.md` を参照

### 表示・操作

- フィルター: 案件タイプ
- フィルターUI: `common-functional-spec.md`「画面・UI共通仕様 / カード共通UI」参照
- 集計期間: タブ共通の集計期間を使用する（`Turnaround/tab-spec.md` 参照）
- 集計対象: `common-functional-spec.md`「集計対象の共通仕様」および `terms-and-definitions.md`「集計対象の設定」参照
- 集計対象の扱い: 本カードの構成分析は主担当案件を対象とする
- 表示形式: `グラフ / テーブル`
- 初期表示: `グラフ`
- ソート項目: `構成分析の合計 / ユーザー名`
- ソート順: `昇順 / 降順`
- 初期ソート: `構成分析の合計-降順`
- ソート適用: グラフとテーブルの両方に適用し、表示切替後も維持する

### グラフ表示

- グラフ種別: 積み上げ横棒グラフ
- X軸: 日数
- Y軸: ユーザー
- セグメント: ステータス分類ごとの日数
- 主+副表示時: 主担当バーを上、副担当バーを下に表示する
- ホバー: セグメントにマウスオーバーしたとき、分類名と日数を表示する
- セグメントクリック: 絞り込み・ドリルダウンを行わない
- ユーザー名クリック: `common-functional-spec.md`「ユーザー詳細Paneへの遷移」参照

### テーブル表示

- カラム構成:
  - `ユーザー名`
  - `合計`（または `合計（主）/ 合計（副）`）
  - 各ステータス分類の日数
  - `初回返信速度（中央値）`
- テーブルUI: `common-functional-spec.md`「テーブルビューの共通ルール」参照
- ユーザー遷移: `common-functional-spec.md`「ユーザー詳細Paneへの遷移」参照

### 集計・データルール

- 構成分析対象: 完了案件のみ（進行中案件は除外）
- 構成分析対象: 主担当案件のみ（副担当案件は除外）
- 算出方法: 各ステータス分類の滞在日数合計を、主担当としての完了案件数で割った平均値を使用する
- 丸め: 小数第1位（0.1日単位）
- 初回返信速度（中央値）定義: `terms-and-definitions.md`「リードタイム/初回返信」参照
- ユーザー・ユーザーグループフィルター: `Turnaround/tab-spec.md` の上部入力定義に従う

### 検証観点

- `TURNAROUND-COMPOSITION-VAL-001` 初期表示はグラフで、ユーザー別・分類別の日数を表示する
- `TURNAROUND-COMPOSITION-VAL-002` セグメントホバー時、分類名と日数を表示する
- `TURNAROUND-COMPOSITION-VAL-003` セグメントクリック時、絞り込みやドリルダウンを実行しない
- `TURNAROUND-COMPOSITION-VAL-004` ユーザー名クリック時、対象ユーザーの Pane（パフォーマンスタブ）を開く
- `TURNAROUND-COMPOSITION-VAL-005` 表示形式切替時、同一集計結果をグラフ/テーブルで表示する
- `TURNAROUND-COMPOSITION-VAL-006` 集計対象が `主+副` でも、構成分析の算出対象は主担当案件のみとする
- `TURNAROUND-COMPOSITION-VAL-007` ソート変更時、グラフ/テーブル双方の並び順を更新し、表示切替後も維持する
