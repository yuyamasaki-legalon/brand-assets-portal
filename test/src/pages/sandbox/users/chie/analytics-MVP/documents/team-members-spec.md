# レポート機能 画面仕様書

本仕様書は「レポート機能」画面の仕様を単独で完結するよう整理したもの。

画面イメージ: https://pr-50-aegis-lab.on-technologies-technical-dept.workers.dev/sandbox/chie/analytics-MVP

## 1. 概要

### Goal

法務部門のマネージャー/担当者が、チーム全体の案件状況を可視化し、適切なアサインと負荷分散を実現し、客観的なパフォーマンス評価と育成計画の立案ができる状態にする。

### 対象ユーザー

- 法務部門のマネージャー/担当者（マターマネジメントのProライセンス保有者）
- 業務: チーム全体の案件管理、負荷分散、パフォーマンス分析

### 課題（As-Is）

- チーム全体の進行中案件の状況を把握するため、各案件を個別に確認する必要がある
- 未割り当て案件のアサイン時、メンバーの状況や経験を感覚で判断せざるを得ない
- メンバー間の負荷の偏りが見えず、適切な再配分ができない
- 納期超過・滞留を早期に発見する仕組みがなく、問題が大きくなるまで気づけない
- 個人のパフォーマンス評価が主観的で、過去データとの比較が困難

### 提供価値とゴール

- **負荷分散**: 誰が忙しいかを可視化し、適切なアサイン調整を支援
- **遅延防止**: 納期逼迫・超過、滞留を早期に発見
- **育成/評価**: 個人の経験（案件タイプ/契約類型）と成果（処理件数・リードタイム等）を客観的に把握
- チーム全体の進行中案件の状況を把握し、担当調整や遅延兆候を早期に検知
- 個人/チームのパフォーマンス推移を確認
- 過去データを分析し、生産性向上・育成・評価に活用

### この画面でできること

- チーム全体の進行中案件を、納期・ステータス・案件タイプ別に可視化し、未割り当て案件のアサインを支援
- 月次の新規案件数・完了案件数の推移を可視化し、メンバー別の負荷状況を比較
- リードタイム（処理速度）の推移を確認し、リードタイムの構成（ステータス分類ごとの日数）を分析
- 特定のメンバーの状況を詳細に確認し、進行中案件一覧やパフォーマンス指標を表示

## 3. 共通コンポーネント

### 3.1 ヘッダー

- 画面タイトル: `レポート`
- 更新ボタン: 現在の業務状況タブのみに表示。押下で画面の表示内容を再取得

### 3.2 タブ

- タブ: `現在の業務状況` / `業務負荷` / `時間`

### 3.3 画面状態の保存方針

この画面は、テーブルの表示設定（カラムの移動/表示非表示/幅調整など）や、各種フィルタ/表示切替が多い。

**保存理由:**
同一タブで案件詳細へ遷移する場合、遷移前の本画面の状態（フィルタ条件/表示切替/表示設定など）を保存し、戻ったときに復元できるようにする必要がある。

**保存先:** ローカルストレージ（ユーザーごと）

**保存対象（基本方針）:**
- テーブルの表示設定（カラム移動/表示非表示/幅調整/固定/ソート）
- フィルタ条件
- 表示切替（表示軸/表示形式（グラフ or テーブル））
- 集計対象の設定（主担当・副担当 / 昨年のデータ）
- ソート設定

各カードの保存内容の詳細は、各カードの仕様書に記載する。

### 3.4 UIコンポーネントパターン

#### フィルターボタン
- 表示: アイコンボタン
- 動作: クリックでPopoverを開き、フィルター項目を選択
- バッジ表示: フィルターがアクティブな場合（いずれかのフィルターが「すべて」以外）に表示、variant: subtle、color: information
- リセットボタン: フィルターがアクティブな場合のみ表示、クリックで全フィルターを「すべて」にリセット

#### 表示形式切替
- 表示: Menuから選択（アイコンボタンで表示、グラフ: LfChartBar、テーブル: LfTable）
- オプション: グラフ / テーブル
- 初期表示: グラフ

#### 表示軸切替
- 表示: Selectコンポーネント
- 配置: Dividerの直後、グラフ表示の前

#### 情報アイコン
- 表示: カードタイトルの横に `LfQuestionCircle` アイコン
- 動作: ホバーでPopoverが自動的に開く、クリックでピン留め/解除
- 内容: カードの説明テキスト

#### 集計期間入力
- 表示: 開始月と終了月を別々に選択（Selectコンポーネント）
- 日付表示形式:
  - 日本語（ja-JP）: `YYYY/M`（例: "2025/1"）
  - 英語（en-US）: `Month YYYY`（例: "Jan 2025"）

#### ユーザー選択
- 表示: TagPickerコンポーネント
- 選択可能: ユーザー（メンバー）とユーザーグループの両方
- 初期状態: 空選択（すべてのユーザーとして扱う）
- 反映範囲: すべてのカードの表示・集計対象に反映
- 空選択時: 案件担当者（主担当）が未割り当ての案件も集計に含む
- ユーザーグループ選択時: そのグループに所属するすべてのユーザーを集計対象とする
- 選択時: 選択したユーザー（ユーザーグループに所属するユーザーを含む）のみを表示対象
- 空選択時: すべてのユーザーを表示対象

#### 集計期間
- 初期値: 直近6ヶ月（今日から6ヶ月前の月初〜今月末）
- 選択可能範囲: **【要相談】** 過去24ヶ月分
- 1回に選択できる範囲: **【エンジニア相談】** 1年分
- 反映範囲: タブ全体（すべてのカード）の表示・集計対象に反映
- 集計期間フィルタの基準日時（指標ごと）: 9.5 参照
- リセットボタン: ユーザー・ユーザーグループまたは集計期間が初期値と異なる場合に表示、クリックで初期値に戻す

#### Checkbox
- 用途: 昨年を表示など
- 動作: ON/OFFで即時に反映
- 昨年を表示: 選択した日付範囲の昨年同期データを表示

### 3.5 集計対象の設定

- オプション: `主担当のみ / 副担当のみ / 主担当・副担当`
- 初期値: 主担当のみ
- 集計単位: メンバーが「案件担当者（主担当）」に設定されている案件数と、「副担当者」に設定されている案件数
- 主/副の扱い: 両方表示時に重複カウントを許可する
- 集計対象の定義:
  - 主担当のみ: 主担当（`assignee`）として紐づく案件のみを集計
  - 副担当のみ: 副担当（`subAssignee`）として紐づく案件のみを集計
  - 主+副: 主担当と副担当を別々に集計し、別々に表示（同一案件が主担当と副担当の両方に紐づく場合は、それぞれ別々にカウント）
- メンバー詳細Paneへの遷移時: 集計対象の設定は引き継がれない（Paneでは常に主担当と副担当の両方を表示）

### 3.6 グラフ表示の共通ルール

#### グラフの高さ・スクロール
- 高さ: メンバー数に応じて動的に計算（最小値240px、最大値640pxが設定されている）
- スクロール: グラフの高さが最大値を超える場合、縦スクロールが可能になる（`overflowY: "auto"`）
- 640px超はスクロール可能

#### バーの間隔
- 主担当と副担当の間、メンバー間に適切な間隔が設定されている

#### 目盛幅の設定
- X軸は非表示、Y軸はメンバー名を表示
- 目盛の最大値は、実際の最大案件数に数値ラベルの表示スペースを考慮した余裕を追加した値（約15%の余裕、最小値は実際の最大値 + 1）
- フィルター（案件タイプ/ステータス/納期）を変更しても目盛幅は変わらない
- 集計対象（主担当のみ/副担当のみ/主+副）を変更したときは目盛幅が変わる

#### Y軸ラベルの表示
- 集計対象が「主+副」または「副担当のみ」の場合: 各メンバーのバーに「主」「副」のラベルを表示
- 集計対象が「主担当のみ」の場合: 「主」「副」のラベルは非表示
- 納期超過の案件があるメンバーは、メンバー名が赤色で表示され、警告アイコンも表示

#### X軸の月ラベルのクリック動作
- クリックで該当月の案件一覧ページを別タブで開く（外部リンクアイコン付き）
- 該当月に完了した案件をフィルターした案件一覧ページを別タブで開く

### 3.7 テーブルビューの共通ルール

#### カラムの表示形式
- 数値は整数
- 小数第1位まで、単位「日」を付与（値が0以下の場合はハイフン（-））
- ヘッダーから「（単位：日）」の表記を削除
- セルの数値は小数第1位まで表示し、単位「日」を付与（例: "4.0 日"）
- 値が0以下の場合はハイフン（-）で表示（単位は非表示）

#### カラムヘッダーの表示ルール
- 主担当のカラム: 集計対象が「主担当のみ」または「主+副」の場合に表示
  - 集計対象が「主担当のみ」の場合: カラム名に「（主）」はつかない
  - 集計対象が「主+副」の場合: カラム名に「（主）」がつく
- 「（副）」がつくカラム: 集計対象が「主+副」または「副担当のみ」の場合に表示

#### 「開く」ボタン
- 表示: 各メンバー行の右端に「開く」ボタンを表示（固定カラムとして右端に配置）
- アイコン: `LfLayoutHorizonRight`をleadingに配置
- ラベル: 「開く」
- クリック: 「開く」ボタンをクリックで、そのメンバーの全案件を表示するPaneを開く
  - 現在のフィルター状態をPaneに引き継ぐ
  - 集計期間も引き継ぐ
  - Pane内のタブは、現在のタブが現在の業務状況タブの場合は進行中の案件タブ、それ以外（業務負荷タブ・時間タブ）の場合はパフォーマンスタブが開く

#### ソート
- ソートキー: 表示中の案件の総件数 / ユーザー名
  - 主担当のみ: 主担当として持っている案件の総件数
  - 副担当のみ: 副担当として持っている案件の総件数
  - 主+副: 上記の合算値（主担当と副担当を別々に集計し、合算）
- デフォルト: 案件数降順（多いほうが上）
- 同率時: ユーザー名の昇順
- 切り替え方法: ボタンからMenuを開き、以下の4つの選択肢から選択:
  - 案件数昇順
  - 案件数降順
  - ユーザー名昇順
  - ユーザー名降順
- 適用範囲: ソート設定はグラフビューとテーブルビューの両方に適用（表示形式を切り替えてもソート順は維持）

#### ソート/表示順（月の並び）
- 月の並び（時系列の昇順: 古い月 → 新しい月）
- 最初に全期間の行を表示し、その後月の並び（時系列の昇順: 古い月 → 新しい月）

### 3.8 メンバー詳細Paneへの遷移

#### グラフビューからの遷移
- 積み上げ棒の各エリアをクリックで、該当条件でフィルターされた状態でメンバー詳細Paneを開く
  - クリックしたエリアに対応するフィルターを設定（ステータス別表示ならステータス、案件タイプ別表示なら案件タイプ、納期別表示なら納期）
  - クリックしたフィルター以外（他のフィルター）は、現在のメインコンテンツのフィルター状態を引き継ぐ
  - 納期別表示の場合、クリックした納期カテゴリに対応する納期フィルターの値を設定（マッピング: `超過` → `納期超過`, `今日` → `今日まで`, `3日以内`（画面上のラベル） → `今日含め3日以内`（フィルター値）, `1週間以内`（画面上のラベル） → `今日含め7日以内`（フィルター値）, `1週間後〜` → `1週間後〜`, `未入力`（画面上のラベル） → `納期未入力`（フィルター値））
  - 業務負荷タブ・時間タブの場合、集計期間も引き継ぐ
  - Pane内のタブの選択: 現在のタブが現在の業務状況タブの場合は進行中の案件タブ、それ以外（業務負荷タブ・時間タブ）の場合はパフォーマンスタブが開く
- メンバー名をクリックで、そのメンバーの全案件を表示するPaneを開く
  - 現在のフィルター状態（案件タイプ / ステータス / 納期フィルター）をPaneに引き継ぐ
  - Pane内のタブは進行中の案件タブが開く

#### テーブルビューからの遷移
- 「開く」ボタンをクリックで、そのメンバーの全案件を表示するPaneを開く
  - 現在のフィルター状態（案件タイプ / ステータス / 納期フィルター）をPaneに引き継ぐ
  - 集計期間も引き継ぐ
  - Pane内のタブは、現在のタブが現在の業務状況タブの場合は進行中の案件タブ、それ以外（業務負荷タブ・時間タブ）の場合はパフォーマンスタブが開く

## 4. 現在の業務状況タブ

### 4.1 上部入力（タブ共通）

- ユーザー・ユーザーグループ（3.4参照）
- 集計対象（3.5参照）
  - 表示: Selectコンポーネント
  - 配置: ユーザー・ユーザーグループフィルターの横
  - オプション: `主担当案件のみ / 副担当案件のみ / 主担当 + 副担当`
  - 初期値: 主担当案件のみ
  - 反映範囲: タブ内全体（未割り当ての案件カード、進行中の案件状況カード）の表示・集計対象に反映
    - メンバーフィルターが適用されている場合、進行中の案件状況カードのy軸（メンバー名）からも該当メンバーが除外される
- リセットボタン
  - 表示: ユーザー・ユーザーグループフィルターまたは集計対象が初期値と異なる場合に表示
  - スタイル: `Button variant="subtle"`
  - 動作: クリックでユーザー・ユーザーグループフィルターを空に、集計対象を「主担当案件のみ」にリセット

### 4.2 未割り当ての案件

- 表示内容: 主担当が空（未割り当て）の案件数
  - 対象ステータス: ステータスが完了のものは除く
  - 未割り当て判定: `assigneeUserId` が `null` の案件
  - 表示形式: カード形式で件数のみ表示
  - ユーザー・ユーザーグループフィルター: タブ共通のフィルター（4.1参照）が適用される
  - 画面状態の保存（3.3 の方針に従う）:
    - ユーザー・ユーザーグループフィルターを保存
    - 集計対象の設定を保存
    - 主担当が空の案件数: 常に表示（件数が0の場合も表示）
    - 副担当が空の案件数: 集計対象の設定に応じて表示
      - 集計対象が「主担当のみ」の場合: 非表示
      - 集計対象が「副担当のみ」または「主+副」の場合: 表示（件数が0の場合も表示）
- 詳細一覧への遷移:
  - ボタンクリックで、案件一覧ページに別タブ遷移
  - 集計対象の設定に応じて、遷移先の一覧の状態が変わる:
    - 集計対象が「主担当のみ」の場合: 案件一覧の未割り当てタブが表示されている状態に遷移（主担当が未設定の案件のみを表示）

## 9. 指標・集計の定義

### 9.1 進行中/完了の定義

- 完了: 案件ステータスが `完了` の案件。完了日時は「ステータス履歴で最後に完了になった日時」を採用（完了→別ステータスへ戻る可能性があるため）
- 進行中: 案件ステータスが `完了` 以外

### 9.2 納期超過の定義

- 納期: 案件に対して設定されている日付（`dueAt`）
- 判定: 日単位
  - 比較対象: `dueAt`（納期）、`baseDate`（基準日。画面表示時の「今日」）
  - 判定: `baseDate` > `dueAt` の場合「超過」、`baseDate` = `dueAt` の場合は「超過」ではない
  - 基準時刻/タイムゾーン: 日付境界の扱いはユーザー設定に従う
- 納期フィルタの判定:
  - 納期未入力: `dueDate`が空（`null`または未設定）
  - 納期内完了: `completionDate` <= `dueDate`（日付単位で比較、時刻は無視）
  - 納期超過: `completionDate` > `dueDate`（日付単位で比較、時刻は無視）

### 9.3 滞留の定義

- 滞留: ステータスに変化がない状態
- 滞留日数の起点: `CaseStatusHistory` の「最後にステータスが変化した日時」から今日まで
- バケット: 0-3日 / 4-7日 / 8-14日 / 15日以上

### 9.4 リードタイム/初回返信

- リードタイム:
  - 集計対象: 現在の案件ステータスが「完了」の案件に限る
  - 起点: `createdAt`（案件作成日時）
  - 終点: `CaseStatusHistory` の「最後に完了になった日時」
  - 集計単位: 完了日時（`completionDate`）の月（`YYYY-MM`）
  - 計算: 該当月に完了した案件のリードタイム（`createdAt`から`completionDate`までの日数）の中央値（配列長が偶数の場合は中央2つの平均）
  - 表示形式: 小数第1位まで（0.1日単位）
  - 欠損: 該当月に完了案件がない場合はハイフン（-）
- 初回返信:
  - 起点: `createdAt`（案件作成日時）
  - 終点:
    - 案件の依頼者（`requesterUserId`）に対して「返信メッセージ」が送られた最初の時点（`CaseMessage`ベース）
  - 対象: 完了案件のうち、初回返信日時（`firstReplyDate`）が存在する案件のみ
  - 集計単位: 完了日時（`completionDate`）の月（`YYYY-MM`）
  - 計算: 該当月に完了した案件の初回返信速度（`createdAt`から`firstReplyDate`までの日数）の中央値（配列長が偶数の場合は中央2つの平均）
  - 表示形式: 小数第1位まで（0.1日単位）
  - 欠損: 該当月に初回返信がある完了案件がない場合はハイフン（-）
- 休日/営業時間補正: 今回は補正しない（休日/営業時間を除外しない）。将来的に営業日/営業時間を除外した指標（営業時間ベース）を採用するかは検討

### 9.5 月次集計の定義

- 集計単位: 月（`YYYY-MM`）
- 集計単位（月次）の境界: ユーザー設定に従う（タイムゾーン/日付境界）
- 集計期間フィルタの基準日時（指標ごと）:
  - 新規案件数: 案件作成日時（`createdAt`）の月で集計
  - 完了案件数: 完了した日時（`completionDate`）の月で集計
  - リードタイム: 完了した日時（`completionDate`）の月で集計
  - 初回返信: 初回返信日時（9.4 の定義に従う）
- 集計期間フィルタの適用: 該当する月が集計期間内に含まれる場合のみ集計
- 新規件数:
  - 対象: すべての案件（ステータスに関係なく）
  - 集計単位: 案件作成日時（`createdAt`）の月（`YYYY-MM`）
  - 集計期間フィルタ: 案件作成日時の月が集計期間内に含まれる場合のみ集計
- 完了件数:
  - 定義: 現在の案件ステータスが完了の案件のうち、「最後にステータスが完了になった日時」が対象月に入る案件数
  - 対象: ステータスが「完了」の案件のみ
  - 完了日時（`completionDate`）: ステータスが最後に「完了」になった日時
  - 集計単位: 完了日時（`completionDate`）の月（`YYYY-MM`）
  - 集計期間フィルタ: 完了日時の月が集計期間内に含まれる場合のみ集計
  - 内訳の分類:
    - 納期内: `completionDate` <= `dueDate`（日付単位で比較、時刻は無視）
    - 納期超過: `completionDate` > `dueDate`（日付単位で比較、時刻は無視）
    - 納期未入力: `dueDate`が空（`null`または未設定）
  - 内訳の扱い:
    - 「納期未入力（`dueAt` が空）」の完了案件も内訳としてカウント
    - 「納期内完了/納期超過完了」の判定は日付単位で行う（9.2 の日単位ルールに従う）
- 欠損/0件の表示: 対象期間に該当データがない月は0で表示
- ソート/表示順: 月の並び（時系列の昇順: 古い月 → 新しい月）
- 代表値:
  - リードタイム/初回返信速度: 中央値
  - 中央値の計算: 配列長が偶数の場合は中央2つの平均
  - **【未決】**構成分析の代表値: 平均/中央値
- **【未決】**中央値の計算対象:
  - リードタイム/初回返信速度の中央値は、該当月に完了（または初回返信が発生）した案件のみを対象にする、でよいか
  - **【未決】**初回返信が存在しない案件の扱い: 中央値の計算からは除外し、別途「初回返信なし件数」を表示（またはツールチップで補足）

- 案件タイプをカスタマイズできるようにして、案件を分類し、その軸で案件を分析
- 遅延リスクの予兆検知とアラート（類似案件の傾向、経過日数、外部要因などの提示）
- 自然言語による分析（例: 「なぜリードタイムが悪化した？」への対話的ドリルダウン）
- 個人の目標/成長の自動トラッキング（未経験領域の可視化、次に挑戦すべき案件の提示）
- チームのスキルマップ可視化とスキルギャップの把握（領域別の偏り、キャリアパスの提案）
- 貢献度の多角的評価（高難易度案件、協力回数、ナレッジ貢献などの指標化）
- 業務改善インサイト（ボトルネック条項/部門別の偏り/手戻り多発の検知、プレイブック更新提案）
- ひな形の効果測定（利用率、リードタイム短縮、交渉回数削減などの可視化）
- 指標（リードタイム/初回返信速度など）の営業時間ベース化（休日/営業時間を除外した計算）
