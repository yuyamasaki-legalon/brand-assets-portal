# Current Workload - Card 1: 未割り当ての案件

## 機能仕様
### カード概要

- 目的: 未割り当て案件の件数を把握し、一覧へ遷移する
- カード名: 未割り当ての案件

### 表示・操作

- 表示形式: カード表示（件数のみ）
- 件数表示: 主担当が空の案件数を表示する
- 集計対象別表示:
  - `主担当のみ`: 主担当が空の件数のみ表示する
  - `副担当のみ / 主+副`: 主担当が空の件数に加え、副担当が空の件数も表示する
- 詳細遷移: ボタンクリックで案件一覧ページを別タブで開く
- 遷移先条件（主担当のみ）: 案件一覧の未割り当てタブを開き、主担当未設定案件のみ表示する
- 遷移先条件（副担当のみ / 主+副）: 上記に加えて `副担当者=未割り当て` を OR 条件で適用する

### 集計・データルール

- 主担当未割り当て判定: `assigneeUserId = null`
- 対象ステータス: `完了` は除外する
- ユーザー・ユーザーグループフィルター:
  - 原則として未割り当て案件は全体対象とする
  - ユーザー・グループ選択時は、未割り当て側に残るもう一方担当（主/副）が選択ユーザーに一致する案件を表示対象に含める
- 件数表示ルール:
  - 0件でも表示する
  - 1000件以上は `999+` 表示とする

### 画面状態の保存

- 保存方針: `common-functional-spec.md`「画面状態の保存ルール」参照
- 保存対象:
  - ユーザー・ユーザーグループフィルター
  - 集計対象

### 検証観点

- `CW-UNASSIGNED-CARD-VAL-001` 件数表示は `0件`、`1件以上`、`1000件以上（999+）` を正しく表示する
- `CW-UNASSIGNED-CARD-VAL-002` `主担当のみ` では主担当未設定のみ表示し、副担当未設定は表示しない
- `CW-UNASSIGNED-CARD-VAL-003` `副担当のみ / 主+副` では主担当未設定と副担当未設定を表示する
- `CW-UNASSIGNED-CARD-VAL-004` ユーザー・ユーザーグループフィルター適用時、未割り当て案件は「全体対象 + もう一方担当一致条件」で判定する
- `CW-UNASSIGNED-CARD-VAL-005` 一覧遷移時、集計対象に応じた条件（主担当未設定のみ / 副担当未設定OR追加）を引き継ぐ
