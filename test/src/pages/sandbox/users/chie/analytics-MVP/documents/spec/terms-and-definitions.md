
# Terms and Definitions（用語・定義）

## 用語・定義


| 用語 | 定義 |
| --- | --- |
| 進行中案件 | 案件ステータスが「完了」以外の案件 |
| 完了案件 | 案件ステータスが「完了」の案件。完了日時は「ステータス履歴で最後に完了になった日時」を採用 |
| 納期超過 | `baseDate`（基準日、画面表示時の「今日」） > `dueAt`（納期）の場合 |
| 納期内完了 | `completionDate` <= `dueDate`（日付単位で比較、時刻は無視） |
| 納期超過完了 | `completionDate` > `dueDate`（日付単位で比較、時刻は無視） |
| 納期未入力 | `dueAt`が空（`null`または未設定） |
| 滞留 | ステータスに変化がない状態。滞留日数の起点は`CaseStatusHistory`の「最後にステータスが変化した日時」から今日まで |
| リードタイム | 案件作成日時（`createdAt`）から完了日時（`completionDate`）までの日数。完了案件のみを対象とし、該当月に完了した案件の中央値で集計 |
| 初回返信日時 | `CaseMessage`から導出。案件の依頼者（`requesterUserId`）に対して「返信メッセージ」が送られた最初の時点 |
| 初回返信速度 | 案件作成日時（`createdAt`）から初回返信日時までの日数。完了案件のうち初回返信日時が存在する案件のみを対象とし、該当月に完了した案件の中央値で集計 |
| 主担当 | 案件担当者（`assignee`）として紐づく案件 |
| 副担当 | 副担当者（`subAssignee`）として紐づく案件 |
| 未割り当ての案件 | 担当種別に応じて、主担当未設定・副担当未設定のいずれかまたは両方を指す。いずれも「案件にユーザーがアサインされていない状態」。進行中案件のみが対象（完了は除く）。担当種別ごとの意味は「集計対象の設定」を参照 |
| 集計期間 | 月次集計の対象期間。集計単位は月（`YYYY-MM`） |
| 構成分析 | リードタイムの構成（ステータス分類ごとの日数）を分析。主担当の案件のみを集計対象とする |
| 集計範囲 | 表示・集計の対象となるデータの範囲。閲覧可能な案件、（ユーザーフィルター適用時は）選択したユーザー・グループに属するユーザー、および期間フィルター（集計期間）で絞り込まれた範囲の集合。「集計範囲に〜が存在する」はこの範囲を指す。なお、未割り当ての案件は「案件にユーザーがアサインされていない状態」であるため、未割り当ての案件の集計範囲にはユーザーフィルターは適用されない |
| 表示対象のユーザー | Proライセンスを保有するユーザー。ユーザーフィルターが設定されている場合は、選択したユーザー・グループに属するユーザーに絞り込まれた結果を指す。カードの表示・集計で「ユーザー別」とある場合はこのユーザーを指す |
| 集計対象（設定） | 画面上の「集計対象」フォームで選ぶ値。担当種別は3通りあり、正式定義は下記「集計対象の設定」を参照。「集計対象が『<担当種別>』に設定されている」はこの設定値を指す |
| 納期区分 | 進行中の「表示軸・納期別」で使う区分と納期フィルターの選択肢は1対1ではない（ラベル・数が異なる）。正式定義は下記「納期区分（表示軸・フィルター）」を参照 |


## 納期区分（表示軸・フィルター）

- 進行中案件の「表示軸・納期別」で使う区分と、納期フィルターの選択肢は**1対1で対応していない**。表示軸は画面上のラベルで6区分、フィルターは「すべて」を除いて6選択肢であり、ラベルも異なる。以下に両方の正式定義と対応関係を記載する。

- **共通**: 基準日（`baseDate`）は画面表示時の「今日」。日付境界・タイムゾーンはユーザー設定に従う。納期（`dueAt`）は日付まで（時刻は持たない）。比較は日単位で行う。

- **表示軸「納期別」の区分（画面上のラベル・6区分）**

| 区分（画面上のラベル） | 境界（日単位） |
| --- | --- |
| 納期超過 | `dueAt` が `baseDate` より前 |
| 今日 | `dueAt` = `baseDate`（同日） |
| 1,2日後 | `dueAt` が `baseDate+1日` から `baseDate+2日` まで（1〜2日後） |
| 3-6日後 | `dueAt` が `baseDate+3日` から `baseDate+6日` まで（3〜6日後） |
| 1週間後〜 | `dueAt` が `baseDate+7日` 以上 |
| 未入力 | `dueAt` が空（`null` または未設定） |

- **納期フィルターの選択肢（7つ）**

| フィルター値 | 境界（日単位） |
| --- | --- |
| すべて | 絞り込みなし |
| 納期未入力 | `dueAt` が空 |
| 納期超過 | `baseDate` > `dueAt` |
| 今日まで | `dueAt` = `baseDate` |
| 今日含め3日以内 | `dueAt` が `baseDate` から `baseDate+2日` まで（0〜2日、今日を含む） |
| 今日含め7日以内 | `dueAt` が `baseDate` から `baseDate+6日` まで（0〜6日、今日を含む） |
| 1週間後〜 | `dueAt` が `baseDate+7日` 以上 |

- **完了案件の納期区分**

- 完了案件の内訳で使う「納期内」「納期超過」「納期未入力」は、本節の進行中向け区分とは別である。定義は用語表の「納期内完了」「納期超過完了」「納期未入力」に従う。


## 集計対象の設定（画面の「集計対象」フォーム・担当種別の正式定義）

- 画面の「集計対象」は次のとおり定義する。

- **オプション（3通り）**: 主担当のみ / 副担当のみ / 主担当・副担当
- **初期値**: 主担当のみ
- **集計単位**: ユーザーが「案件担当者（主担当）」に設定されている案件数と、「副担当者」に設定されている案件数
- **主/副の扱い**: 両方表示時に重複カウントを許可する
- **集計対象の定義（担当種別ごと）**:
   - **主担当のみ**: 主担当（`assignee`）として紐づく案件のみを集計。未割り当ての案件は主担当未設定（`assigneeUserId` が `null`）の案件
   - **副担当のみ**: 副担当（`subAssignee`）として紐づく案件のみを集計。未割り当ての案件は副担当未設定の案件
   - **主+副**: 主担当と副担当を別々に集計し、別々に表示（同一案件が主担当と副担当の両方に紐づく場合は、それぞれ別々にカウント）。未割り当ての案件は主担当未設定または副担当未設定の案件
- **ユーザー詳細Paneへの遷移時**: 集計対象の設定は引き継がれない（Paneでは常に主担当と副担当の両方を表示）

## 指標・集計の定義


### 進行中案件・完了案件の定義

- 完了: 案件ステータスが `完了` の案件。完了日時は「ステータス履歴で最後に完了になった日時」を採用（完了→別ステータスへ戻る可能性があるため）
- 進行中: 案件ステータスが `完了` 以外


### 納期超過の定義

- 納期: 案件に対して設定されている日付（`dueAt`）
- 判定: 日単位
   - 比較対象: `dueAt`（納期）、`baseDate`（基準日。画面表示時の「今日」）
   - 判定: `baseDate` > `dueAt` の場合「超過」、`baseDate` = `dueAt` の場合は「超過」ではない
   - 基準時刻/タイムゾーン: 日付境界の扱いはユーザー設定に従う
- 納期フィルタの判定:
   - 納期未入力: `dueDate`が空（`null`または未設定）
   - 納期内完了: `completionDate` <= `dueDate`（日付単位で比較、時刻は無視）
   - 納期超過: `completionDate` > `dueDate`（日付単位で比較、時刻は無視）


### 滞留の定義

- 滞留: ステータスに変化がない状態
- 滞留日数の起点: `CaseStatusHistory` の「最後にステータスが変化した日時」から今日まで
- バケット: 0-3日 / 4-7日 / 8-14日 / 15日以上


### リードタイム・初回返信の定義

- リードタイム:
   - 集計対象: 現在の案件ステータスが「完了」の案件に限る
   - 起点: `createdAt`（案件作成日時）
   - 終点: `CaseStatusHistory` の「最後に完了になった日時」
   - 集計単位: 完了日時（`completionDate`）の月（`YYYY-MM`）
   - 計算: 該当月に完了した案件のリードタイム（`createdAt`から`completionDate`までの日数）の中央値（配列長が偶数の場合は中央2つの平均）
   - 表示形式: 小数第1位まで（0.1日単位）
   - 欠損: 該当月に完了案件がない場合はハイフン（-）
- 初回返信:
   - 起点: `createdAt`（案件作成日時）
   - 終点:
     - 案件の依頼者（`requesterUserId`）に対して「返信メッセージ」が送られた最初の時点（`CaseMessage`ベース）
   - 対象: 完了案件のうち、初回返信日時（`firstReplyDate`）が存在する案件のみ
   - 集計単位: 完了日時（`completionDate`）の月（`YYYY-MM`）
   - 計算: 該当月に完了した案件の初回返信速度（`createdAt`から`firstReplyDate`までの日数）の中央値（配列長が偶数の場合は中央2つの平均）
   - 表示形式: 小数第1位まで（0.1日単位）
   - 欠損: 該当月に初回返信がある完了案件がない場合はハイフン（-）
- 休日/営業時間補正: 今回は補正しない（休日/営業時間を除外しない）。将来的に営業日/営業時間を除外した指標（営業時間ベース）を採用するかは検討


### 月次集計の定義

- 集計単位: 月（`YYYY-MM`）
- 集計単位（月次）の境界: ユーザー設定に従う（タイムゾーン/日付境界）
- 集計期間フィルタの基準日時（指標ごと）:
   - 新規案件数: 案件作成日時（`createdAt`）の月で集計
   - 完了案件数: 完了した日時（`completionDate`）の月で集計
   - リードタイム: 完了した日時（`completionDate`）の月で集計
   - 初回返信: 初回返信日時（本ファイル「リードタイム/初回返信」の定義に従う）
- 集計期間フィルタの適用: 該当する月が集計期間内に含まれる場合のみ集計
- 新規件数:
   - 対象: すべての案件（ステータスに関係なく）
   - 集計単位: 案件作成日時（`createdAt`）の月（`YYYY-MM`）
   - 集計期間フィルタ: 案件作成日時の月が集計期間内に含まれる場合のみ集計
- 完了件数:
   - 定義: 現在の案件ステータスが完了の案件のうち、「最後にステータスが完了になった日時」が対象月に入る案件数
   - 対象: ステータスが「完了」の案件のみ
   - 完了日時（`completionDate`）: ステータスが最後に「完了」になった日時
   - 集計単位: 完了日時（`completionDate`）の月（`YYYY-MM`）
   - 集計期間フィルタ: 完了日時の月が集計期間内に含まれる場合のみ集計
   - 内訳の分類:
     - 納期内: `completionDate` <= `dueDate`（日付単位で比較、時刻は無視）
     - 納期超過: `completionDate` > `dueDate`（日付単位で比較、時刻は無視）
     - 納期未入力: `dueDate`が空（`null`または未設定）
   - 内訳の扱い:
     - 「納期未入力（`dueAt` が空）」の完了案件も内訳としてカウント
     - 「納期内完了/納期超過完了」の判定は日付単位で行う（本ファイル「納期超過/納期内の判定（日単位）」に従う）
- 欠損/0件の表示: 対象期間に該当データがない月は0で表示
- ソート/表示順: 月の並び（時系列の昇順: 古い月 → 新しい月）
- 代表値:
   - リードタイム/初回返信速度: 中央値
   - 中央値の計算: 配列長が偶数の場合は中央2つの平均
   - **【未決】**構成分析の代表値: 平均/中央値
- **【未決】**中央値の計算対象:
   - リードタイム/初回返信速度の中央値は、該当月に完了（または初回返信が発生）した案件のみを対象にする、でよいか
   - **【未決】**初回返信が存在しない案件の扱い: 中央値の計算からは除外し、別途「初回返信なし件数」を表示（またはツールチップで補足）
