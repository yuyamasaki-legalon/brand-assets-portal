# チーム・メンバー機能（追加機能）実装要求ドキュメント（AI駆動開発用）

---

## 1. 背景/目的（ユーザー価値）

この機能（チーム・メンバー分析）は、法務部門のマネージャー/担当者が以下を達成することを支援する。

- 負荷分散: 誰が忙しいかを把握し、アサイン調整の判断材料を得ながら、案件の担当者をアサインする
- 遅延防止: 納期逼迫・超過、滞留を早期に発見する
- 育成/評価: 個人の経験（案件タイプ/契約類型）と成果（処理件数・リードタイム等）を把握する

### 1.1. 対象ユーザー（本PRD内の定義）

- 本PRD内で「ユーザー」と言う場合、マターマネジメントのProライセンスを持つユーザーを指す
  - 本ページのチーム集計・表示対象となる「メンバー」の母集団は、このProライセンス保有ユーザーである
  - 表示・集計対象の案件は 6.1 の認可/権限制約（閲覧可能な案件のみ）に従う

## 2. 主要ユースケース

### 2.1. ユースケースA: 今の危険信号を見つけ、対応したい（モニタリング）

このユースケースは、チーム全体の「今」の状況をリアルタイムに把握し、日々の業務を最適化することを目的とする。

- **チーム全体の進行中案件の把握**
  - 納期の近さ/超過でざっくり把握し、遅延・炎上を早期に発見する
  - 納期超過、納期当日、納期が近い案件を優先的に確認できる
  - 納期未入力の案件も可視化し、適切な納期設定を促す

- **未割り当て案件のアサイン**
  - チームメンバーの現在の状況（進行中件数、納期逼迫/超過、滞留など）を確認する
  - 各メンバーが担当してきた案件タイプの傾向を参考にする
  - 負荷の偏りを考慮しながら、最適な担当者を判断して未割り当て案件の主担当をアサインする
  - 副担当者も必要に応じてアサインし、チーム全体の負荷分散を図る

- **問題の特定とドリルダウン**
  - 納期超過/逼迫・滞留が多いメンバーを特定する
  - 該当メンバーや該当案件へドリルダウンして詳細を確認できる
  - ボトルネックの特定（電子契約の署名待ちなど、自部門外の要因で停滞している案件）を行う
  - 問題が大きくなる前に対処するための具体的なアクションを判断する

### 2.2. ユースケースB: 仕事量の偏りを見たい（負荷分散）

このユースケースは、チーム全体の仕事量を可視化して、業務の偏りをなくし、適切に仕事を再配分することを目的とする。

- **メンバー別の負荷状況の比較**
  - メンバー別に、担当案件数（主担当/副担当）の内訳を比較する
  - 主担当のみ、副担当のみ、または主+副の合計で負荷を把握できる
  - 各メンバーの現在の負荷状況を一目で比較し、忙しいメンバーを特定する

- **負荷の原因の特定**
  - どのステータス（未着手、確認中、2次確認中、自部門外確認中など）が負荷の原因かを把握する
  - どの案件タイプ（契約書審査、契約書起案、法務相談、その他）が負荷の原因かを把握する
  - 滞留状況（経過日数）も考慮して、単なる件数だけでなく、実際の負荷の重さを評価する

- **再配分の判断**
  - 負荷の偏りを発見したら、適切なアサイン調整の判断材料として活用する
  - 負荷が高いメンバーから、負荷が低いメンバーへ案件を再配分する判断を支援する
  - 個人の状況把握として、必要に応じて特定のメンバーの状況にドリルダウンし、個別の負荷状況や担当案件を確認する

### 2.3. ユースケースC: 過去の傾向を見たい（分析・評価）

このユースケースは、過去のデータを分析し、チームの生産性向上とメンバーの育成・評価に役立てることを目的とする。

- **チーム全体の傾向把握**
  - 月次の新規案件数/完了案件数の推移を確認し、業務量の季節変動トレンドを把握する
  - 昨年度の実績と比較して、チーム全体の処理件数や対応スピードがどう変化したかを定点観測する
  - リードタイム、初回返信速度などの指標の推移を確認し、生産性の変化を把握する
  - 業務量の波を把握し、将来の人員配置や採用計画に役立てる

- **個人のパフォーマンス評価**
  - 個人の指標を、期間・案件タイプ等で絞り込める
  - 担当案件数、リードタイム、対応した契約の複雑さといった客観的なデータに基づき、個人のパフォーマンスを評価する
  - 1on1ミーティングや人事評価の場で、公平で納得感のあるフィードバックを行うための材料とする

- **育成支援**
  - 各メンバーがどのような種類の案件を経験してきたかを把握する
  - チーム全体のスキルセットを向上させるための育成計画や、適切な案件のアサインに活用する
  - 未経験領域の可視化や、次に挑戦すべき案件の提示に活用する

## 3. 画面仕様

この章では「画面で何ができるか」を、実装・テスト可能な粒度で定義する。

### 3.1. 画面構成

- メインタブ
  - `進行中`（モニタリング/負荷分散、未割り当てのアサイン）
  - `業務負荷`（業務量の分析、メンバー別の負荷比較）
  - `時間`（リードタイムの推移と構成分析）
- メンバー詳細Pane
  - メンバーを選ぶと、画面右側のPaneに該当メンバーの詳細（進行中案件一覧 + 指標）を表示できる

### 3.1.1. 画面状態（表示設定/フィルタ）の保存方針

この画面は、テーブルの表示設定（カラムの移動/表示非表示/幅調整など）や、各種フィルタ/表示切替が多い。

**保存する理由:**
同一タブで案件詳細へ遷移する場合、遷移前の本画面の状態（フィルタ条件/表示切替/表示設定など）を保存し、戻ったときに復元できるようにする必要がある。

**保存先:** ローカルストレージ（ユーザーごと）

**保存対象（基本方針）:**
- テーブルの表示設定（カラム移動/表示非表示/幅調整/固定/ソート）
- フィルタ条件
- 表示切替（表示軸/表示形式（グラフ or テーブル））
- 集計対象の設定（主担当・副担当 / 昨年のデータ）
- ソート設定

各カードの保存内容の詳細は、各カードの仕様書に記載する。


### 3.2. 進行中タブ

#### A. 概要（何を見るタブか）

- チーム全体の“進行中”の案件を中心に、納期・滞留・内訳を見て、個人へドリルダウンする

#### B. 表示項目（最低限）
- 迷ったら「表示対象」「集計単位」「操作」「例外（空/権限/欠損）」の順で書く。

##### B-1. 未割り当ての案件

- 主担当が空（未割り当て）
  - 対象ステータス: ステータスが完了のものは除く
  - 未割り当て判定: `assigneeUserId` が `null` の案件
  - 表示形式: カード形式で件数のみ表示
  - 画面状態の保存（3.1.1 の方針に従う）:
    - 集計対象の設定を保存する
    - 主担当が空の案件数: 常に表示される（件数が0の場合も表示される）
    - 副担当が空の案件数: 集計対象の設定に応じて表示される
      - 集計対象が「主担当のみ」の場合: 表示されない
      - 集計対象が「副担当のみ」または「主+副」の場合: 表示される（件数が0の場合も表示される）
    - 両方が表示される場合: 主担当が空の案件数と副担当が空の案件数が縦線（Divider）で区切られて表示される
  - 詳細一覧への遷移:
    - 「案件を見る」ボタンをクリックすると、サービス本体の案件一覧ページが別タブで開く
    - 集計対象の設定に応じて、遷移先の一覧の状態が変わる:
      - 集計対象が「主担当のみ」の場合: 案件一覧の未割り当てタブが表示されている状態に遷移する（主担当が未設定の案件のみを表示）
        - 遷移先URL: `https://app.legalon-cloud.com/case?case-search-condition-definition-type=unassigned&case-search-condition-definition-id=01990991-f52d-706c-a03f-e9887f2138ef&page=1&main-assignee-ids%5B%5D=unassigned&logical-operators%5B%5D=%7B%22fields%22%3A%5B%22main-assignee-ids%22%2C%22sub-assignee-ids%22%5D%2C%22logical-operator%22%3A%22and%22%7D&case-status-ids%5B%5D=...&search-sort-condition=last-message-time-desc&excluding-closed-case=true`
      - 集計対象が「副担当のみ」または「主+副」の場合: 案件一覧の未割り当てタブで副担当者: 未割り当てがor条件で含まれている状態に遷移する（主担当が未設定または副担当が未設定の案件を含む）
        - 遷移先URL: `https://app.legalon-cloud.com/case?case-search-condition-definition-type=unassigned&case-search-condition-definition-id=01990991-f52d-706c-a03f-e9887f2138ef&page=1&main-assignee-ids%5B%5D=unassigned&sub-assignee-ids%5B%5D=unassigned&logical-operators%5B%5D=%7B%22fields%22%3A%5B%22main-assignee-ids%22%2C%22sub-assignee-ids%22%5D%2C%22logical-operator%22%3A%22or%22%7D&due-date-relative=next%2C1%2Cweeks&case-create-time-relative=past%2C3%2Cmonths&last-message-time-relative=past%2C1%2Cweeks&case-status-ids%5B%5D=...&search-sort-condition=last-message-time-desc&excluding-closed-case=true`

##### B-2. 進行中の案件状況

- 目的: メンバー別の案件内訳を確認する
- カードタイトル: 「進行中の案件状況」
  - タイトルの横に情報アイコン（`LfQuestionCircle`）が表示される
  - 情報アイコンにマウスオーバーまたはクリックすると、Popoverが開き、カードの説明が表示される
- メンバー別の案件内訳
  - フィルター: 納期 / ステータス / 案件タイプ
    - 初期値: すべて「すべて」（フィルター無しの状態）
    - フィルターボタン: アイコンボタンで表示される
      - フィルターがアクティブな場合（いずれかのフィルターが「すべて」以外）: ボタンのvariantがsubtle、カラーがinformation、Badgeが表示される
      - フィルターボタンをクリックすると、Popoverが開き、フィルター項目（納期、ステータス、案件タイプ）を選択できる
      - リセットボタン: フィルターがアクティブな場合のみ表示される
        - リセットボタンをクリックすると、すべてのフィルター（納期、ステータス、案件タイプ）が「すべて」にリセットされる
    - 納期フィルター: `すべて / 納期未入力 / 納期超過 / 今日まで / 今日含め3日以内 / 今日含め7日以内 / 1週間後〜`
      - 納期フィルターの区分の境界（日単位）:
        - 納期超過: `dueAt` が `基準日（今日）` より前
        - 今日まで: `dueAt` が `基準日（今日）` と同日（=0日）
        - 今日含め3日以内: `dueAt` が `基準日（今日）` から `基準日（今日）+2日` まで（0〜2日、今日を含む）
        - 今日含め7日以内: `dueAt` が `基準日（今日）` から `基準日（今日）+6日` まで（0〜6日、今日を含む）
        - 1週間後〜: `dueAt` が `基準日（今日）+7日以上`
        - 納期未入力: 案件に付与されている納期（`dueAt`）が空
      - 基準日（今日）: テナントの設定（US/JP）に従う（日付境界/タイムゾーン）
      - 納期（`dueAt`）: 日付まで（時刻は持たない）
  - 集計対象: 主担当のみ / 副担当のみ / 主+副
    - 初期値: 主担当のみ
  - 集計単位: メンバーが「案件担当者（主担当）」に設定されている案件数と、「副担当者」に設定されている案件数
  - 主/副の扱い: 両方表示時に重複カウントを許可する
  - 表示軸: `納期別 / ステータス別 / 案件タイプ別`
    - デフォルト: 納期別
    - 切り替え方法: Selectコンポーネントで選択
    - 表示軸別の分類方法:
      - 納期別: 納期（`dueAt`）に基づいて案件を分類する。区分と区分の境界は以下の通り（画面上のラベル）:
      - 納期超過: `dueAt` が `基準日（今日）` より前
      - 今日: `dueAt` が `基準日（今日）` と同日（=0日）
      - 3日以内: `dueAt` が `基準日（今日）+1日` から `基準日（今日）+2日` まで（1〜2日後）
      - 4-7日以内: `dueAt` が `基準日（今日）+3日` から `基準日（今日）+6日` まで（3〜6日後）
      - 1週間後〜: `dueAt` が `基準日（今日）+7日以上`
      - 未入力: 案件に付与されている納期（`dueAt`）が空
      - ステータス別: テナントに設定されているステータス名をそのまま出す
      - 案件タイプ別: サービスで規定している案件タイプをそのまま出す
  - 表示形式: `グラフ` / `テーブル`
    - 初期表示: グラフ
    - 切り替え方法: Menuから選択（アイコンボタンで表示、グラフの場合はLfChartBarアイコン、テーブルの場合はLfTableアイコン）
  - ソート:
    - ソートキー: 表示中の案件の総件数 / ユーザー名
      - 主担当のみ: 主担当として持っている案件の総件数
      - 副担当のみ: 副担当として持っている案件の総件数
      - 主+副: 上記の合算値
    - デフォルト: 案件数降順（多いほうが上）
    - 同率時: ユーザー名の昇順
    - 切り替え方法: ボタンからMenuを開き、以下の4つの選択肢から選択:
      - 案件数昇順
      - 案件数降順
      - ユーザー名昇順
      - ユーザー名降順
    - 適用範囲: ソート設定はグラフビューとテーブルビューの両方に適用される（表示形式を切り替えてもソート順は維持される）
  - グラフの表示:
    - グラフの高さ: メンバー数に応じて動的に計算される（最小値240px、最大値640pxが設定されている）
    - グラフのスクロール: グラフの高さが最大値を超える場合、縦スクロールが可能になる（`overflowY: "auto"`）
    - バーの間隔: 主担当と副担当の間、メンバー間に適切な間隔が設定されている
    - 目盛幅の設定:
      - X軸は非表示、Y軸はメンバー名を表示
      - 目盛の最大値は、実際の最大案件数に数値ラベルの表示スペースを考慮した余裕を追加した値
      - フィルター（案件タイプ/ステータス/納期）を変更しても目盛幅は変わらない
      - 集計対象（主担当のみ/副担当のみ/主+副）を変更したときは目盛幅が変わる
    - Y軸ラベルの表示:
      - 集計対象が「主+副」または「副担当のみ」の場合: 各メンバーのバーに「主」「副」のラベルが表示される
      - 集計対象が「主担当のみ」の場合: 「主」「副」のラベルは表示されない
      - 納期超過の案件があるメンバーは、メンバー名が赤色で表示され、警告アイコンも表示される
    - グラフの右横に表示する数字:
      - 各積み上げ棒の各セグメント（ステータス/案件タイプ/納期区分ごと）に数字が表示される
      - 各メンバーの合計案件数が右端に表示される（最後のステータス/案件タイプ/納期区分のバーの右横）
      - 合計案件数が0の場合は表示されない
    - ツールチップ:
      - 積み上げ棒の各エリアにマウスオーバーすると、該当の案件名が表示される
      - ステータス別の場合: ツールチップに「（納期：納期ラベル）案件名」の形式で表示される
      - 案件タイプ別の場合: ツールチップに「（納期：納期ラベル）案件名」の形式で表示される
      - 納期別の場合: ツールチップに「（ステータス名）案件名」の形式で表示される
  - メンバー詳細Paneを開く際の共通ルール:
    - 集計対象（主担当のみ / 副担当のみ / 主+副）の設定は引き継がれない（Paneでは常に主担当と副担当の両方が表示される）
  - グラフビュー:
    - グラフのインタラクション:
      - マウスオーバー: 積み上げ棒の各エリアにマウスオーバーすると、ツールチップが表示される
        - ツールチップの内容:
          - メンバー名
          - 各セグメントの名前と値（案件数）
          - 案件名のリスト（「案件」ラベル付き、最大200pxの高さでスクロール可能、各案件名は「・案件名」の形式で表示される）
        - 表示形式:
          - ステータス別の場合: 各ステータスの案件数と、該当する案件名のリストが表示される
          - 案件タイプ別の場合: 各案件タイプの案件数と、該当する案件名のリストが表示される
          - 納期別の場合: 各納期区分の案件数と、該当する案件名のリストが表示される
      - クリック: 積み上げ棒の各エリアをクリックすると、そのエリアに対応する条件でフィルターされた状態でメンバー詳細Paneが開く
        - クリックしたエリアに対応するフィルターが設定される（ステータス別表示ならステータス、案件タイプ別表示なら案件タイプ、納期別表示なら納期）
        - クリックしたフィルター以外（他のフィルター）は、現在のメインコンテンツのフィルター状態が引き継がれる
        - 納期別表示の場合、クリックした納期カテゴリに対応する納期フィルターの値が設定される（マッピング: `超過` → `納期超過`, `今日` → `今日まで`, `3日以内`（画面上のラベル） → `今日含め3日以内`（フィルター値）, `1週間以内`（画面上のラベル） → `今日含め7日以内`（フィルター値）, `1週間後〜` → `1週間後〜`, `未入力`（画面上のラベル） → `納期未入力`（フィルター値））
        - 業務負荷タブ・時間タブの場合、集計期間も引き継がれる
        - Pane内のタブの選択: 現在のタブが進行中タブの場合は進行中の案件タブ、それ以外（業務負荷タブ・時間タブ）の場合はパフォーマンスタブが開く
      - メンバー名のクリック: メンバー名をクリックすると、そのメンバーの全案件を表示するPaneが開く

### 3.3. 業務負荷タブ

#### A. 概要（何を見るタブか）

- チーム全体の業務負荷を期間別・メンバー別に分析し、完了案件数や初回返信速度などの指標を可視化する
- 月次の新規案件数・完了案件数の推移を確認し、どのくらい新規案件が月に来ているのかを把握する
- 完了案件数の内訳（納期内完了/納期超過完了/納期未入力）を確認し、新規案件が納期に間に合うようにどの程度処理できているのかを把握する
- メンバー別の完了案件数の内訳（納期内完了/納期超過完了/納期未入力）と初回返信速度を比較し、メンバー別に業務の偏りがないかを確認する

#### B. 上部入力（タブ共通）

- ユーザー・ユーザーグループ（複数選択）
  - 入力形式: TagPickerコンポーネント
  - 選択可能な項目: ユーザー（メンバー）とユーザーグループの両方を選択可能
  - 初期状態: 空選択（すべてのユーザーとして扱う）
  - 集計への反映有無: 反映する
  - 反映範囲:
    - すべてのカード（新規案件カード、完了案件カード、完了案件（メンバー別）カード）の表示・集計対象に反映する
    - 空選択の扱い: すべてのユーザーとして扱う（=フィルタ無し）
      - ユーザー・ユーザーグループがフィルタされていない状態（空選択）のときは、案件担当者（主担当）が未割り当ての案件も集計に含む
    - ユーザーグループを選択した場合: そのグループに所属するすべてのユーザーを集計対象とする
- 集計期間（開始月/終了月）
  - 入力形式: 開始月と終了月を別々に選択（Selectコンポーネント）
    - 日付表示形式: ロケールに応じて異なる
      - 日本語（ja-JP）: 「YYYY/M」形式（例: "2025/1"）
      - 英語（en-US）: 「Month YYYY」形式（例: "Jan 2025"）
  - 初期値: 直近6ヶ月（今日から6ヶ月前の月初〜今月末）
  - 選択可能範囲: **【要相談】** 過去24ヶ月分
  - 1回に選択できる範囲: **【エンジニア相談】** 1年分（開始月から終了月までの期間が最大1年まで）
    - 開始月と終了月の差が1年（12ヶ月）を超える場合、自動的に調整される
    - 開始月を選択した際、既に終了月が選択されている場合は、開始月から1年以内の範囲のみ選択可能
    - 終了月を選択した際、既に開始月が選択されている場合は、開始月から1年以内の範囲のみ選択可能
  - 集計への反映有無: 反映する
  - 反映範囲: 業務負荷タブ全体（すべてのカード）の表示・集計対象に反映する
  - 集計期間フィルタの基準日時（指標ごと）:
    - 新規案件数: 案件作成日時（`createdAt`、プロトタイプでは`startDate`）の月で集計
    - 完了案件数: 完了した日時（`completionDate`）の月で集計
    - リードタイム: 完了した日時（`completionDate`）の月で集計
  - リセット: ユーザー・ユーザーグループ/期間の入力をクリアする（初期値に戻す）

#### C. カード: 新規案件

    - バーの上に合計数を表示: 各月の新規案件数の合計がバーの上に整数で表示される（合計数も整数表記）

          3. 各案件タイプの完了案件数: 案件タイプの順番（契約書審査 / 契約書起案 / 法務相談 / その他）で表示
        - 集計対象が「副担当のみ」の場合:
          1. メンバー名（name）: 固定カラム（左端に固定）、右端に「開く」ボタンが表示される
          2. 合計（副）: すべての案件タイプの完了案件数の合計（副担当のみ）
          3. 各案件タイプの完了案件数（副）: 案件タイプの順番（契約書審査 / 契約書起案 / 法務相談 / その他）で表示
        - 集計対象が「主+副」の場合:
          1. メンバー名（name）: 固定カラム（左端に固定）、右端に「開く」ボタンが表示される
          2. 合計（主）: すべての案件タイプの完了案件数の合計（主担当のみ）
          3. 合計（副）: すべての案件タイプの完了案件数の合計（副担当のみ）
          4. 各案件タイプの完了案件数（主→副の順）: 案件タイプの順番（契約書審査 / 契約書起案 / 法務相談 / その他）で表示
            - 各案件タイプごとに「主→副」の順で表示される
            - 例: 契約書審査（主）→契約書審査（副）→契約書起案（主）→契約書起案（副）→...
      - カラムの表示形式: 数値は整数で表示される
      - カラムヘッダーの表示ルール:
        - 主担当のカラム: 集計対象が「主担当のみ」または「主+副」の場合に表示される
          - 集計対象が「主担当のみ」の場合: カラム名に「（主）」はつかない
          - 集計対象が「主+副」の場合: カラム名に「（主）」がつく
        - 「（副）」がつくカラム: 集計対象が「主+副」または「副担当のみ」の場合に表示される
          - 集計対象が「副担当のみ」の場合: すべてのカラム名に「（副）」がつく
          - 集計対象が「主+副」の場合: 副担当のカラム名に「（副）」がつく
- 操作:
  - グラフビューのインタラクション:
    - マウスオーバー: 積み上げ棒の各エリアにマウスオーバーすると、ツールチップが表示される
      - 納期別の場合: メンバー名、納期別の完了案件数（納期内/納期超過/納期未入力）、完了案件数の合計
      - 案件タイプ別の場合: メンバー名、案件タイプ別の完了案件数（契約書審査/契約書起案/法務相談/その他）、完了案件数の合計
    - メンバー名のクリック: メンバー名をクリックすると、そのメンバーの全案件を表示するPaneが開く
      - 現在のフィルター状態（案件タイプ / 納期フィルター）がPaneに引き継がれる
      - 集計期間も引き継がれる
      - Pane内のタブはパフォーマンスタブが開く
  - テーブルビューのインタラクション:
    - テーブル表示時は、各メンバー行の右端に「開く」ボタンが表示される（固定カラムとして右端に配置）
      - アイコン: `LfLayoutHorizonRight`をleadingに配置
      - ラベル: 「開く」
    - クリック: 「開く」ボタンをクリックすると、そのメンバーの全案件を表示するPaneが開く
      - 現在のフィルター状態（案件タイプ / 納期フィルター）がPaneに引き継がれる
      - 集計期間も引き継がれる
      - Pane内のタブはパフォーマンスタブが開く
  - 並び替え（ソート）:
    - ソートキー: 表示中の案件の総件数 / ユーザー名
      - 主担当のみ: 主担当として持っている案件の総件数
      - 副担当のみ: 副担当として持っている案件の総件数
      - 主+副: 主担当と副担当の合算値（主担当と副担当を別々に集計し、合算）
    - デフォルト: 案件数降順（多いほうが上）
    - 同率時: ユーザー名の昇順
    - 切り替え方法: ボタンからMenuを開き、以下の4つの選択肢から選択:
      - 案件数昇順
      - 案件数降順
      - ユーザー名昇順
      - ユーザー名降順
  - 画面状態の保存（3.1.1 の方針に従う）:
    - フィルタ条件（案件タイプ / 納期 / 集計期間）を保存する
    - 集計対象の設定を保存する
    - 表示切替（納期別 / 案件タイプ別）を保存する
    - 表示形式（グラフビュー / テーブルビュー）を保存する
    - ソート設定（案件数ソート / ユーザー名ソート、昇順/降順）を保存する

- 集計/表示ルール（確定）:
  - 対象案件:
    - 完了案件数の集計:
      - 対象: ステータスが「完了」の案件のみ集計対象とする
      - 完了日時（`completionDate`）の定義: ステータスが最後に「完了」になった日時を`completionDate`とする
      - 集計単位: 完了日時（`completionDate`）の月（`YYYY-MM`）
      - 集計期間フィルタの適用: 完了日時の月が集計期間内に含まれる場合のみ集計
  - 案件タイプフィルタの適用:
    - フィルタが「すべて」以外の場合、選択した案件タイプの案件のみを集計対象とする
  - 納期フィルタの適用:
    - フィルタが「すべて」以外の場合、以下の条件でフィルタリング:
      - 納期未入力: `dueDate`が空（`null`または未設定）
      - 納期内完了: `completionDate` <= `dueDate`（日付単位で比較、時刻は無視）
      - 納期超過: `completionDate` > `dueDate`（日付単位で比較、時刻は無視）
  - 集計対象（主担当・副担当 / 主担当のみ / 副担当のみ）の適用:
    - 主担当のみ: 主担当（`assignee`）として紐づく案件のみを集計
    - 副担当のみ: 副担当（`subAssignee`）として紐づく案件のみを集計
    - 主+副: 主担当と副担当を別々に集計し、別々に表示する
      - 主担当としての案件数と副担当としての案件数を別々に集計し、別々に表示
      - 同一案件が主担当と副担当の両方に紐づく場合は、それぞれ別々にカウントする
  - 完了案件数の内訳（納期別ビュー）:
    - 納期内: `completionDate` <= `dueDate`（日付単位で比較、時刻は無視）
    - 納期超過: `completionDate` > `dueDate`（日付単位で比較、時刻は無視）
    - 納期未入力: `dueDate`が空（`null`または未設定）
  - 完了案件数の内訳（案件タイプ別ビュー）:
    - 契約書審査 / 契約書起案 / 法務相談 / その他 の4種類で分類
  - ユーザー・ユーザーグループフィルタの適用:
    - ユーザーまたはユーザーグループが選択されている場合、選択したユーザー（ユーザーグループに所属するユーザーを含む）のみを表示対象とする
    - ユーザー・ユーザーグループが空選択の場合、すべてのユーザーを表示対象とする
  - グラフの表示:
    - グラフの高さ: メンバー数に応じて動的に計算される（最小値と最大値が設定されている）
    - バーの間隔: 主担当と副担当の間、メンバー間に適切な間隔が設定されている
    - 目盛幅の設定:
      - X軸は非表示、Y軸はメンバー名を表示
      - 目盛の最大値は、実際の最大案件数に数値ラベルの表示スペースを考慮した余裕を追加した値（約15%の余裕を追加、最小値は実際の最大値 + 1）
      - フィルター（案件タイプ/納期/集計期間）を変更しても目盛幅は変わらない
      - 集計対象（主担当のみ/副担当のみ/主+副）を変更したときは目盛幅が変わる


### 3.4. 時間タブ

#### A. 概要（何を見るタブか）

- チーム全体のリードタイム（処理速度）の推移を確認し、リードタイムの構成（ステータス分類ごとの日数）を分析する
- メンバー別のリードタイム構成を比較し、必要に応じて個人へドリルダウンする

#### B. 上部入力（タブ共通）

- ユーザー・ユーザーグループ（複数選択）
  - 入力形式: TagPickerコンポーネント
  - 選択可能な項目: ユーザー（メンバー）とユーザーグループの両方を選択可能
  - 初期状態: 空選択（すべてのユーザーとして扱う）
  - 集計への反映有無: 反映する
  - 反映範囲:
    - すべてのカード（案件の対応時間カード、案件の完了までの時間の内訳カード）の表示・集計対象に反映する
    - 空選択の扱い: すべてのユーザーとして扱う（=フィルタ無し）
    - ユーザーグループを選択した場合: そのグループに所属するすべてのユーザーを集計対象とする
- 集計期間（開始月/終了月）
  - 入力形式: 開始月と終了月を別々に選択（Selectコンポーネント）
    - 日付表示形式: ロケールに応じて異なる
      - 日本語（ja-JP）: 「YYYY/M」形式（例: "2025/1"）
      - 英語（en-US）: 「Month YYYY」形式（例: "Jan 2025"）
  - 初期値: 直近6ヶ月（今日から6ヶ月前の月初〜今月末）
  - 選択可能範囲: **【要相談】** 過去24ヶ月分
  - 1回に選択できる範囲: **【エンジニア相談】** 1年分（開始月から終了月までの期間が最大1年まで）
    - 開始月と終了月の差が1年（12ヶ月）を超える場合、自動的に調整される
    - 開始月を選択した際、既に終了月が選択されている場合は、開始月から1年以内の範囲のみ選択可能
    - 終了月を選択した際、既に開始月が選択されている場合は、開始月から1年以内の範囲のみ選択可能
  - 集計への反映有無: 反映する
  - 反映範囲: 時間タブ全体（すべてのカード）の表示・集計対象に反映する
  - 集計期間フィルタの基準日時（指標ごと）:
    - リードタイム: 完了した日時（`completionDate`）の月で集計
    - 初回返信速度: 初回返信日時（`firstReplyDate`）の月で集計
    - リードタイム構成分析: 完了日時（`completionDate`）の月で集計（タブ全体の集計期間フィルターと同じ基準）
  - リセットボタン: ユーザー・ユーザーグループまたは集計期間が初期値と異なる場合に表示される
    - 表示条件: ユーザー・ユーザーグループが選択されている、または集計期間が初期値（直近6ヶ月）と異なる場合
    - 動作: クリックすると、ユーザー・ユーザーグループを空選択に戻し、集計期間を初期値（直近6ヶ月）に戻す
    - 配置: 集計期間のSelectコンポーネントの後

#### C. カード: 案件の対応時間

- 目的: チーム全体の月次リードタイム・初回返信速度の推移を俯瞰する
- 入力UI:
  - 情報アイコン（Popover）: カードタイトルの横に情報アイコン（`LfQuestionCircle`）が表示される
    - アイコンをホバーすると、Popoverが自動的に開き、カードの説明が表示される
    - アイコンをクリックすると、Popoverがピン留めされ、再度クリックで解除される
    - Popoverの内容: 月次データの説明テキスト（`t("monthlyDescription")`）
  - 集計期間の表示: カードタイトルの下に集計期間が表示される
    - 表示形式: 日本語の場合「YYYY/M - YYYY/M」、英語の場合「Month YYYY - Month YYYY」
    - 表示条件: 集計期間（開始月・終了月）が両方設定されている場合のみ表示される
  - 昨年を表示: CheckboxでON/OFFできる
    - 選択した日付範囲の昨年同期データを点線で表示する
  - フィルター: 案件タイプ
    - フィルターボタン: アイコンボタンで表示される
      - フィルターがアクティブな場合（案件タイプが「すべて」以外）: ボタンのvariantがsubtle、カラーがinformation、Badgeが表示される
      - フィルターボタンをクリックすると、Popoverが開き、案件タイプを選択できる
      - リセットボタン: フィルターがアクティブな場合のみ表示される
        - リセットボタンをクリックすると、案件タイプフィルターが「すべて」にリセットされ、表示項目（リードタイム中央値と初回返信速度中央値）が両方ONになる
  - 表示形式: グラフビュー / テーブルビュー
    - 初期表示: グラフビュー
    - 切り替え方法: Menuから選択
- 表示:
  - 全期間のカード表示: グラフビューの場合、グラフの上に選択した集計期間全体のリードタイム（中央値）と初回返信速度（中央値）をカード形式で表示
    - グラフビューの時のみ表示される（テーブルビューの時は非表示）
    - 昨年を表示している場合、各カードに昨年のデータも表示される（今年の値の下に表示、例: "1.0 日" / "1.2 日（昨年）"）
  - 月次の複合グラフで表示する（日数系: 線）
    - リードタイム（中央値）と初回返信速度（中央値）を線グラフで表示
    - グラフ上の数字表示: 今年の数字（リードタイム中央値と初回返信速度中央値）は常にグラフ上に表示する
    - 昨年を表示している場合でも、今年の数字は表示される
    - 昨年の数字はグラフ上に表示しない（点線で表示されるが、数字は表示されない）
  - 凡例/ツールチップを表示する
    - 昨年を表示している場合、今年のデータに「（今年）」を追加（例: 「完了までの時間（今年）」「初回返信（今年）」）
    - 昨年のデータには「（昨年）」を表示（例: 「完了までの時間（昨年）」「初回返信（昨年）」）
- 操作:
  - 昨年を表示のON/OFFは、即時にグラフへ反映する
  - フィルターを開閉できる
  - クリック時の挙動: 凡例クリック以外に、線/点をクリックしたときの絞り込み/ドリルダウンは行わない
  - 画面状態の保存（3.1.1 の方針に従う）:
    - フィルタ条件（案件タイプ）を保存する
    - 昨年を表示のON/OFFを保存する
    - 表示形式（グラフビュー / テーブルビュー）を保存する

- 集計/表示ルール（確定）:
  - リードタイム（中央値）の集計:
    - 対象: 完了案件のみ（`completionDate`が存在する案件）
    - 集計単位: 完了日時（`completionDate`）の月（`YYYY-MM`）
    - 計算方法: 該当月に完了した案件のリードタイム（`createdAt`から`completionDate`までの日数）の中央値
    - 中央値の計算: 該当月のリードタイムの配列をソートし、中央値を算出（配列長が偶数の場合は中央2つの平均）
    - 表示形式: 小数第1位まで表示（0.1日単位）
    - 欠損の表示: 中央値が算出できない場合（該当月に完了案件がない場合）はハイフン（-）で表示する
  - 初回返信速度（中央値）の集計:
    - 対象: 完了案件のうち、初回返信日時（`firstReplyDate`）が存在する案件のみ
    - 集計単位: 完了日時（`completionDate`）の月（`YYYY-MM`）
    - 計算方法: 該当月に完了した案件の初回返信速度（`createdAt`から`firstReplyDate`までの日数）の中央値
    - 中央値の計算: 該当月の初回返信速度の配列をソートし、中央値を算出（配列長が偶数の場合は中央2つの平均）
    - 表示形式: 小数第1位まで表示（0.1日単位）
    - 欠損の表示: 中央値が算出できない場合（該当月に初回返信がある完了案件がない場合）はハイフン（-）で表示する
  - 昨年データの集計:
    - 集計期間: 選択した集計期間の1年前の期間（開始月-1年 〜 終了月-1年）
    - 集計ロジック: 今年のデータと同じ（ただし、期間を1年前にシフト）
    - データが存在しない場合: 今年のデータの80%〜120%の範囲でランダムに生成したダミーデータを表示（プロトタイプのみ）
  - 欠損/0件の表示:
    - 対象期間に該当データがない月の扱い: 0で表示する
  - ソート/表示順: 月の並び（時系列の昇順: 古い月 → 新しい月）
  - テーブルビューの仕様:
    - 表示項目（カラムの順番）:
      1. 月（name）: 固定カラム（左端に固定）
      2. リードタイム中央値: 該当月のリードタイム（中央値）
        - 昨年を表示している場合、カラム名は「完了までの時間（今年）」
        - 昨年を表示していない場合、カラム名は「完了までの時間」
      3. リードタイム中央値（昨年）: 昨年を表示している場合のみ表示される
        - カラム名は「完了までの時間（昨年）」
      4. 初回返信速度中央値: 該当月の初回返信速度（中央値）
        - 昨年を表示している場合、カラム名は「初回返信（今年）」
        - 昨年を表示していない場合、カラム名は「初回返信」
      5. 初回返信速度中央値（昨年）: 昨年を表示している場合のみ表示される
        - カラム名は「初回返信（昨年）」
    - カラムの表示形式:
      - ヘッダーから「（単位：日）」の表記を削除
      - セルの数値は小数第1位まで表示し、単位「日」を付与（例: "4.0 日"）
      - 値が0以下の場合はハイフン（-）で表示（単位は表示しない）
    - 行の構成:
      - 全期間のデータ: 選択した集計期間全体のリードタイム中央値と初回返信速度中央値を表示（行のnameは「全期間」）、最初の行に表示
      - 月別のデータ: 各月のリードタイム中央値と初回返信速度中央値を表示
    - ソート/表示順: 最初に全期間の行を表示し、その後月の並び（時系列の昇順: 古い月 → 新しい月）

- **【未決】**決めること:
  - 指標の定義/期間フィルタの基準日時/未決事項は 7.5 に集約（7.5参照）

#### D. カード: 案件の完了までの時間の内訳

> **注記**: 案件数・初回返信の表示は業務負荷タブのE（完了案件（メンバー別）、3.3.E）を参照してください。本カードではリードタイムの構成分析に焦点を当てます。

- 目的: メンバー別に、リードタイムの構成（ステータス分類ごとの日数）を把握する
- 入力UI:
  - フィルターアイコン: Popoverを開くアイコンボタン
    - バッジ表示条件: 案件タイプフィルターが「すべて」以外、または集計期間がタブ全体の初期値（直近6ヶ月）と異なる場合にバッジを表示
    - バッジの色: information
  - 案件タイプフィルタ: このカードにのみ適用する（Popover内のフィルタアイコンから設定）
  - 集計期間（開始月/終了月）: Popover内に配置（案件タイプフィルタの下）
    - 入力形式: 開始月と終了月を別々に選択（Selectコンポーネント）
    - 日付表示形式: ロケールに応じて異なる
      - 日本語（ja-JP）: 「YYYY/M」形式（例: "2025/1"）
      - 英語（en-US）: 「Month YYYY」形式（例: "Jan 2025"）
    - 初期値: タブ全体の集計期間フィルターと同じ（直近6ヶ月）
    - 選択可能範囲: 過去24ヶ月分
    - 1回に選択できる範囲: 1年分（開始月から終了月までの期間が最大1年まで）
      - 開始月と終了月の差が1年（12ヶ月）を超える場合、自動的に調整される
      - 開始月を選択した際、既に終了月が選択されている場合は、開始月から1年以内の範囲のみ選択可能
      - 終了月を選択した際、既に開始月が選択されている場合は、開始月から1年以内の範囲のみ選択可能
    - 集計への反映: タブ全体の集計期間フィルターに反映される（カード専用のフィルターではなく、タブ全体に効くフィルター）
    - 集計期間フィルタの基準日時: 完了日時（`completionDate`）の月で集計（タブ全体の集計期間フィルターと同じ基準）
  - 集計対象（主担当・副担当 / 主担当のみ / 副担当のみ）: ButtonGroupの後にSelectコンポーネントとして配置
    - 初期値: 主担当のみ
    - 注記: 構成分析では主担当固定（本カード内の「集計/表示ルール（確定）」に従う）
  - ステータス分類設定:
    - 分類設定ダイアログを開ける（Popover内のボタンから開く）
  - リセットボタン: 案件タイプフィルターまたは集計期間が初期値と異なる場合に表示
    - 表示条件: 案件タイプフィルターが「すべて」以外、または集計期間がタブ全体の初期値（直近6ヶ月）と異なる場合
    - 動作: クリックすると、案件タイプフィルターを「すべて」に戻し、集計期間をタブ全体の初期値（直近6ヶ月）に戻す
    - 配置: フィルターPopoverの外、ButtonGroupの後
  - 表示形式: グラフビュー / テーブルビュー
    - 初期表示: グラフビュー
  - ビューモード（グラフビューのみ）: グルーピング済み / ステータス別
    - 初期値: グルーピング済み
    - 配置場所: Dividerの直後、グラフ表示の前
    - UI形式: Selectコンポーネント（variant="outline", size="medium"）
    - オプション:
      - グルーピング済み: ステータスを「未着手/作業中/他者待ち」にグルーピングして表示
      - ステータス別: 各ステータス（未着手、確認中、2次確認中、自部門外確認中）を個別に積み上げて表示
- 表示:
  - 構成分析:
    - グルーピング済みビュー:
      - メンバー別に、ステータス分類ごとの代表値（日数）を積み上げ棒で表示する
      - ステータス分類: 未着手 / 作業中 / 他者待ち（主担当）、作業中 / 待機中（副担当）
    - ステータス別ビュー:
      - メンバー別に、各ステータス（未着手、確認中、2次確認中、自部門外確認中）ごとの代表値（日数）を積み上げ棒で表示する
      - ステータス分類設定の「非表示」に該当するステータスは表示しない
      - 各ステータスの色はSTATUS_ORDERから取得
    - 軸: 横軸=日数 / 縦軸=メンバー
    - 表示（ラベル/強調）:
      - 納期超過の案件があるメンバーであっても、メンバーラベルを赤くする等の強調は行わない
      - バーの表示順序: 主担当のバーが下、副担当のバーが上に表示される（bothモードの場合）
      - 初回返信速度（中央値）はグラフ上には表示せず、tooltip上にのみ表示される
  - 凡例/ツールチップを表示する
    - tooltipには以下が表示される:
      - 合計（主）: 主担当の合計日数（主+副モードの場合のみ）
      - 合計（副）: 副担当の合計日数（主+副モードの場合のみ）
      - 初回返信速度（中央値）: 初回返信速度の中央値
      - 各ステータス分類ごとの日数
  - テーブルビュー:
    - メンバー名、合計（主担当のみの場合は合計、主+副の場合は合計（主）と合計（副））、各ステータス分類ごとの日数、初回返信速度（中央値）を表示
    - カラムの順番（主+副モードの場合）:
      - 合計（主）→ステータス1（主）→…→合計（副）→ステータス1（副）→…→初回返信速度（中央値）
- 操作:
  - メンバー名のクリックで、メンバー詳細Paneへドリルダウンできる
  - 並び替え（ソート）:
    - ソートキー: 構成分析の合計（積み上げ合計日数） / ユーザー名
    - ソート順: 昇順 / 降順
    - 推奨（初期値）: 構成分析の合計（積み上げ合計日数）の降順
    - 切り替え方法: ボタンからMenuを開き、以下の4つの選択肢から選択:
      - 案件数昇順（構成分析の合計の昇順）
      - 案件数降順（構成分析の合計の降順）
      - ユーザー名昇順
      - ユーザー名降順
  - 画面状態の保存（3.1.1 の方針に従う）:
    - フィルタ条件（案件タイプ）を保存する
    - 集計対象の設定を保存する
    - 表示形式（グラフビュー / テーブルビュー）を保存する
    - ビューモード（グルーピング済み / ステータス別）を保存する
    - ステータス分類設定の一時的な状態を保存する（テナント設定としてDB保存されるが、ユーザーが変更した分類設定の一時的な状態はローカルストレージに保存可能）
    - ソート設定（構成分析の合計ソート / ユーザー名ソート）を保存する
    - 注記: 集計期間はタブ全体のフィルターとして管理されるため、カード単位での保存は行わない

- 集計/表示ルール（確定）:
  - 構成分析の対象:
    - 完了案件のみ含める（進行中案件は含めない）
    - 主担当の案件のみ対象とする（副担当として紐づく案件は含めない）
    - 集計結果の丸め: 小数第1位（0.1日単位）
    - 計算方法: 各ステータス分類ごとに、該当ステータスに滞在していた日数の合計を、主担当としての完了案件数で割った平均値（案件あたりの平均日数）
  - ステータス分類設定（分類ルール）:
    - 分類の粒度と名称:
      - 主担当: 未着手 / 作業中 / 他者待ち / 非表示
      - 副担当: 作業中 / 待機中 / 非表示（副担当に「未着手」相当は持たせない）
    - 追加されたステータスの初期分類:
      - 主担当: 作業中
      - 副担当: 待機中
    - 保存単位: テナント共通の設定
    - 保存先: DB（テナント設定として保存する）
  - 初回返信速度（中央値）の集計（テーブルビューのみ表示）:
    - 対象: 主担当案件のみを対象とする（副担当として紐づく案件は含めない）
    - 計算方法: 該当メンバーが主担当として持つ完了案件の初回返信速度（`createdAt`から`firstReplyDate`までの日数）の中央値
    - 中央値の計算: 初回返信速度の配列をソートし、中央値を算出（配列長が偶数の場合は中央2つの平均）
    - 表示形式: 小数第1位まで表示（0.1日単位）
    - 欠損の表示: 中央値が算出できない場合（該当メンバーに初回返信がある完了案件がない場合）はハイフン（-）で表示する
  - ユーザー・ユーザーグループフィルタの適用:
    - ユーザーまたはユーザーグループが選択されている場合、選択したユーザー（ユーザーグループに所属するユーザーを含む）のみを表示対象とする
    - ユーザー・ユーザーグループが空選択の場合、すべてのユーザーを表示対象とする
  - グラフの表示:
    - グラフの高さ: メンバー数に応じて動的に計算される（最小値と最大値が設定されている）
    - バーの間隔: 主担当と副担当の間、メンバー間に適切な間隔が設定されている
    - 目盛幅の設定:
      - X軸は非表示、Y軸はメンバー名を表示
      - 目盛の最大値は、実際の最大値に数値ラベルの表示スペースを考慮した余裕を追加した値
      - フィルター（案件タイプ/集計期間）を変更しても目盛幅は変わらない
      - 集計対象（主担当のみ/副担当のみ/主+副）を変更したときは目盛幅が変わる
    - Y軸ラベルの表示:
      - 集計対象が「主+副」または「副担当のみ」の場合: 各メンバーのバーに「主」「副」のラベルが表示される
      - 集計対象が「主担当のみ」の場合: 「主」「副」のラベルは表示されない
  - グラフのインタラクション:
    - マウスオーバー: 積み上げ棒の各エリアにマウスオーバーすると、該当のステータス分類名と日数が表示される
    - クリック: 積み上げ棒の各エリアをクリックしたときの絞り込み/ドリルダウンは行わない
    - メンバー名のクリック: メンバー名をクリックすると、そのメンバーの全案件を表示するPaneが開く
      - 現在のフィルター状態（案件タイプフィルター）がPaneに引き継がれる
      - 集計期間も引き継がれる
      - Pane内のタブはパフォーマンスタブが開く
  - テーブルのインタラクション:
    - テーブル表示時は、各メンバー行の右端に「開く」ボタンが表示される（固定カラムとして右端に配置）
      - アイコン: `LfLayoutHorizonRight`をleadingに配置
      - ラベル: 「開く」
    - クリック: 「開く」ボタンをクリックすると、そのメンバーの全案件を表示するPaneが開く
      - 現在のフィルター状態（案件タイプフィルター）がPaneに引き継がれる
      - 集計期間も引き継がれる
      - Pane内のタブはパフォーマンスタブが開く

### 3.5. メンバー詳細Pane

#### A. 概要（何を見るPaneか）

- 選択したメンバーの詳細情報（進行中案件一覧、パフォーマンス指標）を表示する
- メインコンテンツからドリルダウンして、特定のメンバーの状況を詳細に確認する

#### B. 表示位置・構成

- 表示: 画面右のPaneに「メンバー詳細」を表示する
  - Pane内にも `進行中` / `パフォーマンス` のタブがある
  - 初期表示は、メインコンテンツの操作に応じてタブ/フィルタ条件が引き継がれる

#### C. 進行中の案件タブ

- 対象: ステータスが完了以外すべて（未着手も含む）
- 一覧:
  - 主担当としての一覧 / 副担当としての一覧を分けて表示する
    - 主担当一覧: 該当する案件がある場合のみ表示される（0件の場合は非表示）
    - 副担当一覧: 該当する案件がある場合のみ表示される（0件の場合は非表示）
    - 両方とも0件の場合は、EmptyStateが表示される
  - 並び: 納期が空 → 納期（日付）の昇順（早い日付が上）
  - フィルタ: ステータス / 案件タイプ / 納期区分
    - フィルターボタン: ボタンで表示される
      - フィルターがアクティブな場合（いずれかのフィルターが「すべて」以外）: ボタンのvariantがsubtle、カラーがinformation、Badgeが表示される
      - フィルターボタンをクリックすると、Popoverが開き、フィルター項目（納期、ステータス、案件タイプ）を選択できる
      - リセットボタン: フィルターがアクティブな場合のみ表示される
        - リセットボタンをクリックすると、すべてのフィルター（ステータス、案件タイプ、納期区分）が「すべて」にリセットされる
    - ステータス: テナントに設定されているステータス名を全列挙（完了は除く）
    - 納期区分: `すべて / 納期未入力 / 納期超過 / 今日まで / 今日含め3日以内 / 今日含め7日以内 / 1週間後〜`
  - 表示設定（カラム移動/表示非表示/幅調整/固定/ソート、保存の有無）: 3.1.1 の方針に従う
  - 行クリック:
    - テーブルの行をクリックすると、該当の案件詳細ページに遷移する
    - 別タブで開く動線も提供する（案件名列に別タブで開くアイコンボタンが表示される）
    - 画面状態の保存: 3.1.1 に従う
- 表示項目（最小）:
  - 共通: 案件名 / 納期 / ステータス（滞留日数表示付き）
    - 案件名: リンク形式で表示され、クリックすると案件詳細ページに遷移する。別タブで開くアイコンボタンも表示される
    - 納期: 納期超過の場合は、警告アイコンと「超過」ラベルが表示される
    - ステータス: ステータスラベルと、最後にステータスが変化してからの経過日数（滞留日数）が表示される
  - 主担当一覧: 上記に加えて、副担当者名を最後に表示する
  - 副担当一覧: 上記に加えて、案件担当者名（主担当者名）を最後に表示する
- 権限: 6.1 の認可/権限制約に従う（閲覧可能な案件のみ表示、更新権限がある場合のみ編集可能）

#### D. パフォーマンスタブ

- 集計/表示ルール（確定）:
  - 対象ユーザー: 右Paneで選択中のメンバー（=1人）
  - 削除された案件は集計対象に含めない（6.1 に従う）
  - 集計期間フィルタ:
    - メインコンテンツの集計期間フィルタがPaneに引き継がれる
    - 指標ごとの基準日時は、7.5 に従う

##### D-1. カード: 完了案件

- 目的: 個人の月次案件数の推移を確認する
- 入力UI:
  - 情報アイコン（Popover）: カードタイトルの横に情報アイコン（`LfQuestionCircle`）が表示される
    - アイコンをホバーすると、Popoverが自動的に開き、カードの説明が表示される
    - アイコンをクリックすると、Popoverがピン留めされ、再度クリックで解除される
    - Popoverの内容: 個人のパフォーマンス説明テキスト（`t("personalPerformanceDescription")`）
  - 集計期間の表示: カードタイトルの下に集計期間が表示される
    - 表示形式: 日本語の場合「YYYY/M - YYYY/M」、英語の場合「Month YYYY - Month YYYY」
    - 表示条件: 集計期間（開始月・終了月）が両方設定されている場合のみ表示される
  - フィルター: 案件タイプ / 集計期間
    - フィルターボタン: アイコンボタンで表示される
      - フィルターがアクティブな場合（案件タイプが「すべて」以外、または集計期間が初期値と異なる）: ボタンのvariantがsubtle、カラーがinformation、Badgeが表示される
      - フィルターボタンをクリックすると、Popoverが開き、案件タイプと集計期間を選択できる
      - リセットボタン: フィルターがアクティブな場合のみ表示される
        - リセットボタンをクリックすると、案件タイプフィルターが「すべて」にリセットされ、集計期間が初期値（直近6ヶ月）に戻る
  - 表示形式: グラフ（月次棒グラフ） / グラフ（全期間円グラフ） / テーブルビュー
    - 初期表示: グラフ（月次棒グラフ）
    - 切り替え方法: Menuから選択
      - グラフ（月次棒グラフ）: 月次の案件数を棒グラフで表示
      - グラフ（全期間円グラフ）: 選択した集計期間全体の案件数を円グラフで表示
      - テーブルビュー: 月次の案件数をテーブルで表示
  - 表示軸: 納期別 / 案件タイプ別
    - デフォルト: 納期別
    - 切り替え方法: Selectコンポーネントで選択
    - 配置場所: Dividerの直後、グラフ表示の前
- 表示:
  - 月次棒グラフ（納期別ビュー）: 月次の完了案件数を棒グラフで表示
    - 積み上げ棒グラフ: 完了案件数を納期内/納期超過/納期未入力で積み上げて表示
    - 凡例の順番: 納期内 / 納期超過 / 納期未入力の順で表示される
    - バー内の数値表示: 各内訳（納期内/納期超過/納期未入力）の完了案件数がバー内（各セグメント内）に整数で表示される
    - バーの上に合計数を表示: 各月の完了案件数の合計がバーの上に整数で表示される（合計数も整数表記）
  - 月次棒グラフ（案件タイプ別ビュー）: 月次の完了案件数を棒グラフで表示
    - 積み上げ棒グラフ: 案件タイプ別に積み上げて表示（契約書審査 / 契約書起案 / 法務相談 / その他の順）
    - 凡例の順番: 案件タイプの順番（契約書審査 / 契約書起案 / 法務相談 / その他）で表示される
    - バー内の数値表示: 各案件タイプの完了案件数がバー内（各セグメント内）に整数で表示される
    - バーの上に合計数を表示: 各月の完了案件数の合計がバーの上に整数で表示される（合計数も整数表記）
  - 全期間円グラフ（納期別ビュー）: 選択した集計期間全体の新規案件数と完了案件数を納期別の円グラフで表示
    - 新規案件数の円グラフ: 選択した集計期間全体の新規案件数を円グラフで表示
    - 完了案件数の円グラフ: 選択した集計期間全体の完了案件数を納期別（納期内/納期超過/納期未入力）の円グラフで表示
  - 全期間円グラフ（案件タイプ別ビュー）: 選択した集計期間全体の新規案件数と完了案件数を案件タイプ別の円グラフで表示
    - 新規案件数の円グラフ: 選択した集計期間全体の新規案件数を案件タイプ別の円グラフで表示
    - 完了案件数の円グラフ: 選択した集計期間全体の完了案件数を案件タイプ別の円グラフで表示
  - テーブルビュー（納期別）: 月次の完了案件数をテーブルで表示
    - 表示項目（カラムの順番）:
      1. 月（name）: 固定カラム（左端に固定）
      2. 合計: すべての内訳（納期内/納期超過/納期未入力）の完了案件数の合計
      3. 納期内

## 7. 指標・集計の定義

ここは「正解がサービス本体にある」前提なので、プロトタイプ実装の計算は参考に留める。

### 7.1. 進行中/完了の定義

- 完了:
  - 案件ステータスが `完了` の案件
  - 完了日時は「ステータス履歴で最後に完了になった日時」を採用する（完了→別ステータスへ戻る可能性があるため）
- 進行中:
  - 案件ステータスが `未着手` と `完了` 以外

### 7.2. 納期超過の定義

- 納期: 案件に対して設定されている日付（`dueAt`）
- “超過”の判定: 日単位

#### 7.2.1. 日単位の判定ルール

- 比較対象:
  - `dueAt`（納期）
  - `baseDate`（基準日。画面表示時の「今日」）
- 判定:
  - `baseDate` の日付が `dueAt` の日付を過ぎている場合（`baseDate` > `dueAt`）を「超過」とする
  - 同一日（`baseDate` = `dueAt`）の場合は「超過」ではない
  - 基準時刻/タイムゾーン: 日付境界の扱いはテナント設定に従う

### 7.3. 滞留の定義

- 滞留 = ステータスに変化がない状態
- 滞留日数の起点:
  - `CaseStatusHistory` の「最後にステータスが変化した日時」から今日まで
- バケット: 0-3日 / 4-7日 / 8-14日 / 15日以上

### 7.4. リードタイム/初回返信

- リードタイム:
  - 集計対象: 現在の案件ステータスが「完了」の案件に限る
  - 起点: `createdAt`（案件作成日時）
  - 終点: `CaseStatusHistory` の「最後に完了になった日時」
- 初回返信:
  - 起点: `createdAt`（案件作成日時）
  - **【未決】**終点:
    - 候補a: 案件の依頼者（`requesterUserId`）に対して「返信メッセージ」が送られた最初の時点（`CaseMessage`ベース）
    - 候補b: 未着手→他ステータスへの最初の遷移（`CaseStatusHistory`ベース）

- 休日/営業時間補正:
  - 今回は補正しない（休日/営業時間を除外しない）
  - 将来的に、営業日/営業時間を除外した指標（営業時間ベース）を採用するかは検討する（5.4）

### 7.5. 月次集計の定義

- 集計単位: 月（`YYYY-MM`）
- 集計単位（月次）の境界: テナント設定に従う（タイムゾーン/日付境界）
- 集計期間フィルタの基準日時（指標ごと）:
  - 新規案件数: 案件作成日時（`createdAt`）
  - 完了案件数: 完了した日時（「最後にステータスが完了になった日時」）
  - リードタイム: 完了した日時（「最後にステータスが完了になった日時」）
  - 初回返信: 初回返信日時（7.4 の定義に従う）
- 新規件数:
  - `createdAt` の月でカウント
  - 案件ステータスに関係なく、すべての案件を対象とする
- 完了件数:
  - 定義:
    - 候補: 現在の案件ステータスが完了の案件のうち、「最後にステータスが完了になった日時」が対象月に入る案件数
    - 内訳の扱い:
      - 「納期未入力（`dueAt` が空）」の完了案件も内訳としてカウントする
      - 「納期内完了/納期超過完了」の判定は日付単位で行う（7.2 の日単位ルールに従う）
- 代表値:
  - リードタイム/初回返信速度: 中央値
  - **【未決】**構成分析の代表値: 平均/中央値（B-2 の方針に従う）
- **【未決】**中央値の計算対象:
  - リードタイム/初回返信速度の中央値は、該当月に完了（または初回返信が発生）した案件のみを対象にする、でよいか
  - **【未決】**初回返信が存在しない案件の扱い
    - 候補: 中央値の計算からは除外し、別途「初回返信なし件数」を表示する（またはツールチップで補足する）
