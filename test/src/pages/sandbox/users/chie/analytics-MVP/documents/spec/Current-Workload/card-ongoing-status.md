# Current Workload - Card 2: 進行中の案件状況

## 機能仕様
### カード概要

- 目的: ユーザー別の進行中案件の内訳を把握する
- カード名: 進行中の案件状況
- 情報アイコン: `common-functional-spec.md`「画面・UI共通仕様 / カード共通UI」参照
- 集計対象: `Current-Workload/tab-spec.md`「上部入力（タブ共通）」参照

### 表示・操作

- フィルター: `納期 / ステータス / 案件タイプ`
- フィルターUI: `common-functional-spec.md`「画面・UI共通仕様 / カード共通UI」参照
- フィルター初期値: すべて `すべて`
- 納期フィルター項目: `すべて / 納期未入力 / 納期超過 / 今日まで / 今日含め3日以内 / 今日含め7日以内 / 1週間後〜`
- 表示軸: `納期別 / ステータス別 / 案件タイプ別 / 依頼部署別`
- 表示軸初期値: `納期別`
- 表示軸`納期別`の項目: `納期超過 / 今日 / 1,2日後 / 3-6日後 / 1週間後〜 / 未入力`
- 表示軸`ステータス別`: テナント定義のステータス名をそのまま使用
- 表示軸`案件タイプ別`: サービス定義の案件タイプ名をそのまま使用
- 表示軸`依頼部署別`: サービス定義（ユーザーが登録）の部署名をそのまま使用
- 表示形式: `グラフ / テーブル`
- 表示形式初期値: `グラフ`
- ソート項目:
  - `案件数`: `降順 / 昇順`
  - `ユーザー名`: `昇順 / 降順`
- 初期ソート: `案件数-降順`
- ソート適用: グラフ表示・テーブル表示の両方
- ソート保持: 表示形式を切り替えても維持する

### グラフ表示

- グラフ種別: 横向き積み上げ棒グラフ
- 軸: Y軸はユーザー名、X軸は案件数
- セグメント表示: 各セグメントに件数を表示する
- 合計表示: バー右端にユーザー合計を表示する（合計0は非表示）
- ツールチップ共通表示: ユーザー名、セグメント名、件数、案件名リスト（`max-height: 200px` でスクロール）
- ツールチップ（ステータス別/案件タイプ別）: `（納期：納期ラベル）案件名` を表示する
- ツールチップ（納期別）: `（ステータス名）案件名` を表示する
- セグメントクリック: `common-functional-spec.md`「ユーザー詳細Paneへの遷移」参照
- ユーザー名クリック: `common-functional-spec.md`「ユーザー詳細Paneへの遷移」参照

### テーブル表示

- 共通:
  - 左端固定カラムにユーザー名を表示する
  - ユーザー名カラム右端に `開く` ボタンを表示する
  - カラム表示形式・ヘッダー・テーブル操作は `common-functional-spec.md`「テーブルビューの共通ルール」参照
- ステータス別カラム:
  - `主担当のみ`: `合計` + `各ステータス`
  - `副担当のみ`: `合計（副）` + `各ステータス（副）`
  - `主+副`: `合計（主）` + `合計（副）` + `各ステータス（主→副）`
- 案件タイプ別カラム:
  - `主担当のみ`: `合計` + `各案件タイプ`
  - `副担当のみ`: `合計（副）` + `各案件タイプ（副）`
  - `主+副`: `合計（主）` + `合計（副）` + `各案件タイプ（主→副）`
  - 案件タイプ順: `契約書審査 / 契約書起案 / 法務相談 / その他`
- 納期別カラム:
  - `主担当のみ`: `合計` + `各納期区分`
  - `副担当のみ`: `合計（副）` + `各納期区分（副）`
  - `主+副`: `合計（主）` + `合計（副）` + `各納期区分（主→副）`
  - 納期区分順: `未入力 / 納期超過 / 今日 / 1,2日後 / 3-6日後 / 1週間後〜`

### 集計・データルール

- 納期フィルター選択肢: `すべて / 納期未入力 / 納期超過 / 今日まで / 今日含め3日以内 / 今日含め7日以内 / 1週間後〜`
- 納期判定の基準日: `今日`（ユーザー設定のタイムゾーン/日付境界に従う）
- `dueAt` は日付のみ（時刻なし）として判定する
- 納期フィルター区分定義:
  - `納期超過`: `dueAt < 基準日（今日）`
  - `今日まで`: `dueAt = 基準日（今日）`
  - `今日含め3日以内`: `dueAt = 基準日（今日）〜+2日`
  - `今日含め7日以内`: `dueAt = 基準日（今日）〜+6日`
  - `1週間後〜`: `dueAt >= 基準日（今日）+7日`
  - `納期未入力`: `dueAt` が空
- 表示軸`納期別`の区分ラベル: `納期超過 / 今日 / 1,2日後 / 3-6日後 / 1週間後〜 / 未入力`
- 表示軸`納期別`の区分定義:
  - `納期超過`: `dueAt < 基準日（今日）`
  - `今日`: `dueAt = 基準日（今日）`
  - `1,2日後`: `dueAt = 基準日（今日）+1日〜+2日`
  - `3-6日後`: `dueAt = 基準日（今日）+3日〜+6日`
  - `1週間後〜`: `dueAt >= 基準日（今日）+7日`
  - `未入力`: `dueAt` が空
- 表示軸`依頼部署別`: グラフのみ対応（テーブル非表示）

### 画面状態の保存

- 保存方針: `common-functional-spec.md`「画面状態の保存ルール」参照
- 保存対象:
  - フィルター条件（案件タイプ / ステータス / 納期）
  - ユーザー・ユーザーグループフィルター
  - 集計対象
  - 表示軸
  - 表示形式
  - ソート設定

### 検証観点

- `CW-ONGOING-CARD-VAL-001` 初期表示時、表示対象ユーザーが `0件` でも `1件以上` でもカード自体は表示される
- `CW-ONGOING-CARD-VAL-002` フィルター適用時、条件一致案件のみを表示し、該当案件がないユーザー行も表示する
- `CW-ONGOING-CARD-VAL-003` 表示形式を切り替えても、同一の集計結果を保持する
- `CW-ONGOING-CARD-VAL-004` グラフのセグメント選択・ユーザー名選択・テーブルの `開く` 操作は、対象ユーザーのPane（進行中タブ）を開き、現在のフィルター条件を引き継ぐ
- `CW-ONGOING-CARD-VAL-005` 納期超過案件があるユーザーは視覚的に強調表示される
- `CW-ONGOING-CARD-VAL-006` ソート変更時、グラフとテーブルの並び順を同時更新し、表示形式を切り替えても維持する
- `CW-ONGOING-CARD-VAL-007` フィルターリセット実行時、全フィルターを `すべて` に戻し、適用中状態を解除する
