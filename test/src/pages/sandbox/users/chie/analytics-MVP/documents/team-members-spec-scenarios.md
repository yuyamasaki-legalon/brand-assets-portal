## 2. 共通前提（必須）

> ATDD/BDDとTDDの両方で必須となる情報をまとめます。  
> ここに記載した内容は、ユーザーシナリオと機能の振る舞いの双方から参照します。

### 目的

法務部門のマネージャー/担当者が、法務チーム全体の案件状況を可視化し、適切なアサインと負荷分散を実現し、客観的なパフォーマンス評価と育成計画の立案ができる状態にする。法務部門に複数チームがある場合はチーム単位で把握できるようにする。

### 背景

#### 課題（As-Is）
- チーム全体の進行中案件の状況を把握するため、各案件を個別に確認する必要がある
- 未割り当て案件のアサイン時、メンバーの状況や経験を感覚で判断せざるを得ない
- ユーザー間の負荷の偏りが見えず、適切な再配分ができない
- 納期超過・滞留を早期に発見する仕組みがなく、問題が大きくなるまで気づけない
- 個人のパフォーマンス評価が主観的で、過去データとの比較が困難

#### 提供価値とゴール
- **負荷分散**: 誰が忙しいかを可視化し、適切なアサイン調整を支援
- **遅延防止**: 納期逼迫・超過、滞留を早期に発見
- **育成/評価**: 個人の経験（案件タイプ/契約類型）と成果（処理件数・リードタイム等）を客観的に把握
- チーム全体の進行中案件の状況を把握し、担当調整や遅延兆候を早期に検知
- 個人/チームのパフォーマンス推移を確認
- 過去データを分析し、生産性向上・育成・評価に活用

### スコープ

- **含む**:
  - チーム全体の進行中案件を、納期・ステータス・案件タイプ別に可視化し、未割り当て案件のアサインを支援
  - 月次の新規案件数・完了案件数の推移を可視化し、ユーザー別の負荷状況を比較
  - リードタイム（処理速度）の推移を確認し、リードタイムの構成（ステータス分類ごとの日数）を分析
  - 特定のユーザーの状況を詳細に確認し、進行中案件一覧やパフォーマンス指標を表示
  - 画面状態の保存（ローカルストレージ）
  - 権限に基づく表示制御

- **含まない**:
  - AIによる担当者レコメンド付きのアサイン
  - 案件タイプのカスタマイズ機能
  - 遅延リスクの予兆検知とアラート
  - 自然言語による分析
  - 個人の目標/成長の自動トラッキング
  - チームのスキルマップ可視化
  - 貢献度の多角的評価
  - 業務改善インサイト
  - ひな形の効果測定
  - 指標の営業時間ベース化

### 利用環境

- ブラウザ: モダンブラウザ（Chrome、Firefox、Safari、Edgeの最新版）
- 端末: PC
- リージョン: 日本、US

### 権限・ロール

- **閲覧可能なユーザー**: マターマネジメントのProライセンスを保有していれば誰でも閲覧可能。
- **前提権限**:
  - 案件の閲覧権限（閲覧可能な案件のみ表示・集計対象となる）

### 外部依存
- なし

### 初期データ条件

- テナントに登録されているステータス名
- サービスで規定している案件タイプ（契約書審査 / 契約書起案 / 法務相談 / その他）
- ユーザー・ユーザーグループのマスタデータ
- テナントに登録されている部署名
- 案件データ
- ステータス変更履歴データ（`CaseStatusHistory`）
- メッセージデータ（`CaseMessage`、初回返信の判定用）

### 仕様上の制約

- 表示・集計対象は閲覧可能な案件に限定される
- 削除済み案件は表示・集計対象外
- 集計期間の選択可能範囲: テナント作成日からの全データ・ただし1回に選択できる範囲は1年分
- 上部エリアの集計期間フィルターでは、1年を超える範囲は指定できない（開始月〜終了月は最大12ヶ月）
- 集計単位は月（`YYYY-MM`）単位
- 日付境界の扱いはユーザー設定に従う
- 休日/営業時間補正は行わない（将来的に検討）

### 非機能要件（概要）

- **性能**: 画面表示は2秒以内を目標（要計測）
- **可用性**: ブラウザのローカルストレージに依存（データ永続化）
- **セキュリティ**: 権限に基づく表示制御、閲覧可能な案件のみ表示

## 3. 用語・定義（任意）

| 用語 | 定義 |
| --- | --- |
| 進行中案件 | 案件ステータスが「完了」以外の案件 |
| 完了案件 | 案件ステータスが「完了」の案件。完了日時は「ステータス履歴で最後に完了になった日時」を採用 |
| 納期超過 | `baseDate`（基準日、画面表示時の「今日」） > `dueAt`（納期）の場合 |
| 納期内完了 | `completionDate` <= `dueDate`（日付単位で比較、時刻は無視） |
| 納期超過完了 | `completionDate` > `dueDate`（日付単位で比較、時刻は無視） |
| 納期未入力 | `dueAt`が空（`null`または未設定） |
| 滞留 | ステータスに変化がない状態。滞留日数の起点は`CaseStatusHistory`の「最後にステータスが変化した日時」から今日まで |
| リードタイム | 案件作成日時（`createdAt`）から完了日時（`completionDate`）までの日数。完了案件のみを対象とし、該当月に完了した案件の中央値で集計 |
| 初回返信日時 | `CaseMessage`から導出する。案件の依頼者（`requesterUserId`）に対して「返信メッセージ」が送られた最初の時点。 |
| 初回返信速度 | 案件作成日時（`createdAt`）から初回返信日時までの日数。完了案件のうち初回返信日時が存在する案件のみを対象とし、該当月に完了した案件の中央値で集計 |
| 主担当 | 案件担当者（`assignee`）として紐づく案件 |
| 副担当 | 副担当者（`subAssignee`）として紐づく案件 |
| 未割り当ての案件 | 担当種別に応じて、主担当未設定・副担当未設定のいずれかまたは両方を指す。いずれも「案件にユーザーがアサインされていない状態」。進行中案件のみが対象（完了は除く）。担当種別ごとの意味は「集計対象の設定」を参照 |
| 集計期間 | 月次集計の対象期間。集計単位は月（`YYYY-MM`） |
| 構成分析 | リードタイムの構成（ステータス分類ごとの日数）を分析。主担当の案件のみを集計対象とする |
| 集計範囲 | 表示・集計の対象となるデータの範囲。閲覧可能な案件、（ユーザーフィルター適用時は）選択したユーザー・グループに属するユーザー、および期間フィルター（集計期間）で絞り込まれた範囲の集合。「集計範囲に〜が存在する」はこの範囲を指す。なお、未割り当ての案件は「案件にユーザーがアサインされていない状態」であるため、未割り当ての案件の集計範囲にはユーザーフィルターは適用されない |
| 表示対象のユーザー | マターマネジメントのProライセンスを保有するユーザー。ユーザーフィルターが設定されている場合は、選択したユーザー・グループに属するユーザーに絞り込まれた結果を指す。カードの表示・集計で「ユーザー別」とある場合はこのユーザーを指す |
| 集計対象（設定） | 画面上の「集計対象」フォームで選ぶ値。担当種別は3通りあり、正式定義は下記「集計対象の設定」を参照。「集計対象が『<担当種別>』に設定されている」はこの設定値を指す |
| 納期区分 | 進行中の「表示軸・納期別」で使う区分と納期フィルターの選択肢は1対1ではない（ラベル・数が異なる）。正式定義は下記「納期区分（表示軸・フィルター）」を参照 |

### 納期区分（表示軸・フィルター）

進行中案件の「表示軸・納期別」で使う区分と、納期フィルターの選択肢は**1対1で対応していない**。表示軸は画面上のラベルで6区分、フィルターは「すべて」を除いて6選択肢であり、ラベルも異なる。以下に両方の正式定義と対応関係を記載する。

- **共通**: 基準日（`baseDate`）は画面表示時の「今日」。日付境界・タイムゾーンはユーザー設定に従う。納期（`dueAt`）は日付まで（時刻は持たない）。比較は日単位で行う。

**表示軸「納期別」の区分（画面上のラベル・6区分）**

| 区分（画面上のラベル） | 境界（日単位） |
| --- | --- |
| 納期超過 | `dueAt` が `baseDate` より前 |
| 今日 | `dueAt` = `baseDate`（同日） |
| 3日以内 | `dueAt` が `baseDate+1日` から `baseDate+2日` まで（1〜2日後） |
| 4-7日以内 | `dueAt` が `baseDate+3日` から `baseDate+6日` まで（3〜6日後） |
| 1週間後〜 | `dueAt` が `baseDate+7日` 以上 |
| 未入力 | `dueAt` が空（`null` または未設定） |

**納期フィルターの選択肢（7つ）**

| フィルター値 | 境界（日単位） |
| --- | --- |
| すべて | 絞り込みなし |
| 納期未入力 | `dueAt` が空 |
| 納期超過 | `baseDate` > `dueAt` |
| 今日まで | `dueAt` = `baseDate` |
| 今日含め3日以内 | `dueAt` が `baseDate` から `baseDate+2日` まで（0〜2日、今日を含む） |
| 今日含め7日以内 | `dueAt` が `baseDate` から `baseDate+6日` まで（0〜6日、今日を含む） |
| 1週間後〜 | `dueAt` が `baseDate+7日` 以上 |

**完了案件の納期区分**

完了案件の内訳で使う「納期内」「納期超過」「納期未入力」は、本節の進行中向け区分とは別である。定義は用語表の「納期内完了」「納期超過完了」「納期未入力」に従う。

### 集計対象の設定（画面の「集計対象」フォーム・担当種別の正式定義はここに限る）

画面の「集計対象」は次のとおり定義する。

- **オプション（3通り）**: 主担当のみ / 副担当のみ / 主担当・副担当
- **初期値**: 主担当のみ
- **集計単位**: ユーザーが「案件担当者（主担当）」に設定されている案件数と、「副担当者」に設定されている案件数
- **主/副の扱い**: 両方表示時に重複カウントを許可する
- **集計対象の定義（担当種別ごと）**:
  - **主担当のみ**: 主担当（`assignee`）として紐づく案件のみを集計。未割り当ての案件は主担当未設定（`assigneeUserId` が `null`）の案件
  - **副担当のみ**: 副担当（`subAssignee`）として紐づく案件のみを集計。未割り当ての案件は副担当未設定の案件
  - **主+副**: 主担当と副担当を別々に集計し、別々に表示（同一案件が主担当と副担当の両方に紐づく場合は、それぞれ別々にカウント）。未割り当ての案件は主担当未設定または副担当未設定の案件
- **ユーザー詳細Paneへの遷移時**: 集計対象の設定は引き継がれない（Paneでは常に主担当と副担当の両方を表示）

## 5. 機能の振る舞い（TDD向け）

### 5.1 振る舞い仕様テンプレート

#### 5.1.1 集計ロジック

##### BH-001: 進行中案件数の集計

- **振る舞いID**: `BH-001`
- **対象**: 進行中案件数の集計関数
- **責務**: 集計対象の進行中案件数を集計対象設定に応じて集計する
- **参照する共通前提**: 集計対象の設定、権限に基づく表示制御

| 事前状態 | 入力 | 操作 | 期待出力 | 状態変化 |
| --- | --- | --- | --- | --- |
| 集計対象が「主担当のみ」に設定されている | 集計対象の案件データ | 集計実行 | 主担当として紐づく進行中案件数 | なし |
| 集計対象が「副担当のみ」に設定されている | 集計対象の案件データ | 集計実行 | 副担当として紐づく進行中案件数 | なし |
| 集計対象が「主+副」に設定されている | 集計対象の案件データ | 集計実行 | 主担当と副担当を別々に集計した案件数 | なし |

**不変条件**:
- 閲覧可能な案件のみを集計対象とする
- 削除済み案件は集計対象外
- ステータスが「完了」の案件は集計対象外

##### BH-002: リードタイム（中央値）の計算

- **振る舞いID**: `BH-002`
- **対象**: リードタイム計算関数
- **責務**: 完了案件のリードタイム（中央値）を計算する
- **参照する共通前提**: リードタイムの定義、月次集計の定義

| 事前状態 | 入力 | 操作 | 期待出力 | 状態変化 |
| --- | --- | --- | --- | --- |
| 集計期間が設定されている | 完了案件データ、集計期間 | 計算実行 | 該当月のリードタイム中央値（小数第1位まで） | なし |
| 該当月に完了案件が0件 | 完了案件データ、集計期間 | 計算実行 | ハイフン（-） | なし |

**不変条件**:
- 完了案件のみを対象とする
- 集計単位は月（`YYYY-MM`）
- 中央値の計算: 配列長が偶数の場合は中央2つの平均

##### BH-003: 初回返信速度（中央値）の計算

- **振る舞いID**: `BH-003`
- **対象**: 初回返信速度計算関数
- **責務**: 完了案件の初回返信速度（中央値）を計算する
- **参照する共通前提**: 月次集計の定義

**初回返信日時の導出**（生データではなく`CaseMessage`から導出する）:
- **終点**: 案件の依頼者（`requesterUserId`）に対して「返信メッセージ」が送られた最初の時点。`CaseMessage`のうち、依頼者宛ての返信メッセージを時系列で見たときの最初の1件の日時とする。

**初回返信速度の集計**:
- **起点**: `createdAt`（案件作成日時）
- **終点**: 上記の初回返信日時
- **対象**: 完了案件のうち、初回返信日時が導出できる案件のみ（初回返信が存在しない案件は集計対象外）
- **集計単位**: 完了日時（`completionDate`）の月（`YYYY-MM`）
- **計算**: 該当月に完了した案件について、各案件の初回返信速度（`createdAt`から初回返信日時までの日数）を求め、その中央値で集計（配列長が偶数の場合は中央2つの平均）
- **表示形式**: 小数第1位まで（0.1日単位）
- **欠損**: 該当月に初回返信がある完了案件が0件の場合はハイフン（-）で表示

| 事前状態 | 入力 | 操作 | 期待出力 | 状態変化 |
| --- | --- | --- | --- | --- |
| 集計期間が設定されている | 完了案件データ、`CaseMessage`（初回返信日時の導出用）、集計期間 | 計算実行 | 該当月の初回返信速度中央値（小数第1位まで） | なし |
| 該当月に初回返信がある完了案件が0件 | 完了案件データ、`CaseMessage`、集計期間 | 計算実行 | ハイフン（-） | なし |

**不変条件**:
- 完了案件のうち、初回返信日時が導出できる案件のみを対象とする
- 集計単位は月（`YYYY-MM`）
- 中央値の計算: 配列長が偶数の場合は中央2つの平均

| 納期が空（`null`） | 納期（`null`）、基準日（`baseDate`） | 判定実行 | 「納期未入力」 | なし |
