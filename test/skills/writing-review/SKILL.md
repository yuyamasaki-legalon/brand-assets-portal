# UIテキスト ライティングレビュー

UIテキストがサービスUIテキストガイドラインに準拠しているかをレビューするスキル。

## 使用方法

### 明示的な呼び出し
```
/writing-review                          # 現在のdiff対象をレビュー
/writing-review path/to/file.tsx         # 特定ファイルをレビュー
/writing-review src/pages/sandbox/foo/   # ディレクトリ内をレビュー
```

## 実行手順

### Step 1: 対象ファイルの特定

引数 `$ARGUMENTS` からファイルパスを取得する。

- 引数がある場合: そのパスを使用
- 引数がない場合: `git diff` で変更されたファイルを対象

### Step 2: テキストの抽出

対象ファイルからUIテキストを抽出:
- JSX内の文字列リテラル
- `label`, `placeholder`, `title`, `description` などの props
- `children` として渡されるテキスト

### Step 3: ガイドライン照合

抽出したテキストを以下の観点でチェック。
各観点の詳細ルールは `docs/rules/ui/06_writing.md` および参照ドキュメントを確認すること。

1. **トーン＆マナー**: 丁寧な表現、一貫したトーン
2. **用字・用語**: 漢字/ひらがなの使い分け（例: 「ください」「できる」「など」はひらがな推奨）
3. **ボタンラベル**: 動詞形「〜する」または明確な名詞形
4. **エラーメッセージ**: 原因と対処法を含む「〜してください」形式
5. **確認ダイアログ**: 行動の結果が明確な「〜しますか？」形式
6. **プレースホルダー**: 入力例として「例:」で始める
7. **英語テキスト検出**: UIラベルは日本語推奨（技術用語・固有名詞は例外）

### Step 4: 指摘の出力

問題がある箇所を以下の形式で報告:

```
## レビュー結果

### 指摘事項

1. **ファイル**: `src/components/Foo.tsx:42`
   - 現状: "削除出来ません"
   - 推奨: "削除できません"
   - 理由: 「できる」はひらがな表記を推奨

2. **ファイル**: `src/components/Bar.tsx:15`
   - 現状: "保存"
   - 推奨: "保存する"
   - 理由: ボタンラベルは動詞形を推奨
```
