# エッジ抽出ルール

SPEC.md の「インタラクション（Interactions）」節から FlowMap 用のエッジ（`FlowMapEdge`）を抽出するルール。

## パース対象

インタラクション節の各バレット行。形式: `- アクション → 結果`

## 抽出手順

### 1. `→` で分割

各行を `→` で左辺（アクション）と右辺（結果）に分割する。

- 左辺 → エッジの `label`
- 右辺 → ターゲットノードの特定に使用

`→` がない行はスキップ。

### 2. ターゲットノードの特定

右辺のテキストからターゲットノードを推定する。

| 右辺のパターン | ターゲットのノードタイプ | 例 |
|---|---|---|
| `〜Drawerを開く` | drawer | 「詳細Drawerを開く」→ `detail-drawer` |
| `〜Drawerを閉じる` | drawer からの戻り | ソースとターゲットを逆にして `back` エッジ |
| `〜Dialogを表示` | dialog | 「確認Dialogを表示」→ `confirm-dialog` |
| `〜Dialogを閉じる` | dialog からの戻り | ソースとターゲットを逆にして `back` エッジ |
| `〜に遷移` / `〜を開く` | page | 「案件詳細に遷移」→ `case-detail` |
| `〜フロー` | page | 「案件作成フロー」→ `case-create` |

右辺がノード遷移を示さない場合（状態更新のみなど）はエッジを生成しない。

**状態更新のみの例**（エッジ生成しない）:
- `案件情報フォーム変更 → 状態更新`
- `履歴表示Switch切り替え → タイムラインのステータスイベント表示/非表示`

### 3. ソースノードの決定

デフォルトのソースノードは「メインページ」（page タイプの最初のノード）。

以下の場合はソースノードを変更する:

| 条件 | ソースノード |
|---|---|
| 左辺のコンテキストが Drawer 内の操作 | 該当 Drawer ノード |
| 左辺のコンテキストが Dialog 内の操作 | 該当 Dialog ノード |
| 「閉じる」「戻る」系のアクション | 現在の Drawer/Dialog ノード |

**コンテキスト判定**: インタラクション行の直前に「Drawer (xxx) 内:」などのセクションヘッダーがある場合、そのエリア内の操作として扱う。

### 4. エッジ ID の生成

`e-{source}-{target}` の形式で生成する。

同一ソース・ターゲットペアに複数のエッジがある場合は末尾に連番を付与: `e-source-target-2`

## エッジ label の設定

左辺のアクションテキストをそのまま label に使用する。ただし冗長な場合は簡潔にする:

| 元テキスト | label |
|---|---|
| テーブル行クリック | 行クリック |
| フィルターボタンクリック | フィルター |
| 「案件を作成」ボタン | 新規作成 |
| タブ切り替え | タブ切替 |

## {要実装} マーカーの扱い

`{要実装}` が付いたインタラクション行からもエッジを生成する。label の末尾に `(未実装)` を付与する。

**例**: `「案件を作成」ボタン → {要実装: 案件作成フロー}`
→ `{ id: "e-case-list-case-create", source: "case-list", target: "case-create", label: "新規作成 (未実装)" }`

## 重複排除

同一の source-target ペアで同じ label のエッジが複数生成された場合は1つに統合する。
