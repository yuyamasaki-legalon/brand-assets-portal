# ページ間接続の自動検出ルール

サービスレベル集約モード（`--service`）で複数の SPEC.md からノード・エッジを統合する際、ページ間の接続を自動検出するルール。

## 検出方法

### 1. パスベース一致

SPEC A の Interactions セクションに他の SPEC B のルートパス（`path`）が含まれている場合、ページ間エッジを生成する。

```
例:
  SPEC A (case-detail): Interactions に "→ /template/loc/review" と記述
  SPEC B (review): path = "/template/loc/review"
  → エッジ: case-detail → review
```

### 2. 画面名ベース一致

SPEC A の Interactions セクションに他の SPEC B の画面名（`label`）が参照されている場合、ページ間エッジを生成する。

一致判定:
- 完全一致: `レビュー画面` → label `レビュー`
- 部分一致: `案件一覧へ遷移` → label `案件一覧`
- 英語名一致: `navigate to Review` → label `Review` or `レビュー`

### 3. アクション動詞パターン

Interactions 内の以下のパターンを検出:
- `{画面名}へ遷移`
- `{画面名}を開く`
- `{画面名}に移動`
- `navigate to {画面名}`
- `open {画面名}`

## エッジ生成ルール

| 条件 | ソース | ターゲット | ラベル |
|------|--------|-----------|--------|
| パス参照 | 参照元 SPEC のメインページ | 参照先 SPEC のメインページ | Interaction のアクションテキスト |
| 画面名参照 | 参照元 SPEC のメインページ | 参照先 SPEC のメインページ | 参照を含む Interaction テキスト（短縮） |

## ラベル抽出ルール

元の Interaction テキストからエッジラベルを抽出する:

1. アクション動詞 + 対象を抽出: `「レビュー開始」ボタンクリック → レビュー画面へ遷移` → `レビュー開始`
2. 動詞のみの場合はそのまま使用: `クリック` → `クリック`
3. 長すぎる場合は 15 文字で切り詰め

## 重複排除

同じソース・ターゲットペアのエッジが複数検出された場合、最初に検出されたもののみを採用する。
