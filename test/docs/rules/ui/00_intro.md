# はじめに
このドキュメントは、LegalOn Technologiesが提供するプロダクトのUI（ユーザーインターフェース）をデザイン・開発する上で守るべき、基本的な考え方とルールをまとめたガイドラインです。
# 私たちが作るプロダクト
私たちのプロダクトは、toB（企業向け）のSaaSツールであり、特に法律や契約といった専門業務をAIの力でサポートし、業務の効率化と品質の向上を実現します。
ターゲットは特定の業務のプロフェッショナルが中心となりますが、プロダクトによっては企業の全従業員が利用することもあります。そのため、私たちのUIは以下の思想を両立させる必要があります。
- **専門性と平易性:** 専門家の高度な要求に応えつつも、スキルレベルや利用頻度に関わらず「誰でもわかる・誰でも使える」シンプルな体験を提供する。
- **効率化とナレッジ化:** ユーザーの操作負担を最小限に抑え、業務を効率化するだけでなく、使うほどに社内ナレッジがサービスに蓄積され、組織全体の品質向上に貢献するサイクルを生み出す。
- **AIとの自然な協調:** ユーザーがAIの能力を自然に引き出し、その判断根拠を理解し、納得して活用できるインターフェースを目指す。ユーザーの目的を最も簡単な方法で実現することを最優先とする。
# このガイドの目的
本ガイドは、LegalOn Technologies の各プロダクトが一貫した体験品質を保ちながら、効率的に設計・実装・検証できるようにするための共有基盤を提供します。実務で迷った際に「まず参照する一次資料」として機能することを目的とします。
# 対象読者
このガイドラインは、プロダクト開発に関わる以下のメンバーが、それぞれの役割において参照し、実践することを想定しています。
- **プロダクトデザイナー:** 新しいUIを設計する際の、思考の拠り所であり、アウトプットの品質を担保するための指針として。
- **エンジニア:** 実装の仕様を確認し、デザインの一貫性をコードレベルで担保するための、信頼できる情報源として。
- **PdM（プロダクトマネージャー）:** プロダクトの体験価値を定義し、チーム内の意思決定を円滑にするための、共通言語として。
- **QA（品質保証担当者）:** テストの観点を定義し、プロダクトが目指すべき品質基準を判断するための、客観的な基準として。

# 本ガイドの読み方
- **記述形式**:
	- 【ルール】: 遵守すべき必須の規則。
	- （理由）: 直前の【ルール】の背景/目的。
	- 【指針】: 強く推奨されるベストプラクティス。
	- 【定義】: 用語や概念の範囲。
	- 【補足】: 例外/具体例/追加情報。
- **情報源**: 信頼性担保のため、すべての参考リンクは元記事の**一次情報URL**を直接記載します。出典不明時は `MISSING_SOURCE:` を残します。
# ガイドラインの原則と運用
## 原則と逸脱プロセス
このガイドラインは、厳格なルールでデザイナーの創造性を縛るものではなく、プロダクトが従うべき普遍的な「原則」を示すものです。
