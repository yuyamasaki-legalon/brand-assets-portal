# Aegis Component 編集ガイドライン

このファイルは、`aegis_component` ディレクトリ内の各Markdownファイルを編集・作成する際のルールを定めるものです。
Notionページとの同期を円滑に行い、ドキュメントの品質を保つために、以下のルールを守ってください。

## 基本方針
- **対象読者:** このドキュメントは、Aegisデザインシステムの利用者（デザイナー、エンジニア、プロダクトマネージャーなど）を対象とします。
- **明確さ:** 誰が読んでも理解できるよう、専門用語を多用せず、平易な言葉で記述します。
- **簡潔さ:** 不要な修飾語を避け、一文を短く保ちます。
- **一貫性:** Aegisの設計思想に基づき、ページ全体の構成やトーン＆マナーを統一します。
- **実装との連携:** Storybook (Chromatic) へのリンクを積極的に追加し、デザインと実装の乖離を防ぎます。
- **対話を通じた品質向上:** 利用者の提案を歓迎すると同時に、AIアシスタントはデザインシステム全体の整合性やドキュメントの一貫性を保つ観点から、再考を促したり代替案を提示したりすることがあります。これは、最終的なガイドラインの品質を最大化するための協調的なプロセスです。

## ページ構成テンプレート
新しいコンポーネントページを作成する際は、以下の構成を基本としてください。不要なセクションは削除しても構いません。

```markdown

---

## AIアシスタントとの連携ワークフロー
このガイドラインは、AIアシスタントと連携して効率的に作業を進めるための手順を定めます。

### 1. 編集の開始
修正したいファイル名をAIアシスタントに伝えるだけで、面倒な準備作業はAIが自動で行います。
このワークフローの目的は、**Notion上の最新情報との意図しないコンフリクトを避ける**ことです。
- **ユーザー:** 「`Button.md`を修正したい」と指示。
- **AIアシスタント:**
    - 対応するNotionページを特定し、最新の内容を取得します。
    - ローカルのMarkdownファイルと比較し、リッチコンテンツのプレースホルダーなどを追加・修正して、ファイルを編集可能な最新の状態に同期します。

### 2. ガイドラインの編集
AIによる準備が完了したら、ユーザーは本来の編集作業に集中できます。
- 変更したい内容をAIアシスタントに具体的に指示してください。
- **編集の基本原則:**
    - **既存の文章を尊重する:** ガイドラインの更新は、多くの場合、既存の説明に情報を**追記・補足**する形となります。元の文章が持つコンテキストや意図を尊重し、安易に全文を書き換えるのではなく、必要な情報を付け加えるように編集してください。
    - **構造を維持する:** `<columns>`のようなレイアウト構造は、Notionでの表示を意図したものです。この構造を壊さないように、カラム内にテキストやプレースホルダーを追加・編集してください。

### 3. Notionへの反映
編集が完了したら、AIアシスタントにNotionへの反映を指示します。
- **ユーザー:** 「Notionに反映して」と指示。
- **AIアシスタント:** 変更内容をNotionページに反映させる更新処理を試みます。

### 4. 【重要】最終確認と手動更新
AIによる自動更新は、特にFigma埋め込みなどのリッチコンテンツを含むページでは、**レイアウト崩れや失敗のリスク**を伴います。

- **必ずNotionページを目視で確認してください:** AIの「成功しました」という報告があった場合でも、必ずブラウザで実際のNotionページを開き、意図通りに更新されているか、レイアウトが崩れていないかを確認してください。
- **問題があった場合:**
    - **手動で修正:** 最も安全な方法は、変更箇所をMarkdownファイルからコピーし、Notionページに直接ペーストすることです。
    - **バージョン履歴から復元:** Notionのページ履歴機能を使って、問題が発生する前の状態に戻してください。

## テキストの書き方
- **見出し:** `#` (h1), `##` (h2), `###` (h3) を使用します。
- **強調:**
  - `**太字**` を使用します。
  - `💡` はコンポーネント概要やTipsに使用します。
  - `⚠️` は特に注意が必要な警告に使用します。
- **リスト:** 箇条書きは `-` を使用します。
- **コード:** コード片やプロパティ名は `` `code` `` で囲みます。
- **リンク:**
  - 別のAegisコンポーネントへの言及は、可能であればファイルへの相対リンクを記述します。
  - `[Buttonコンポーネント](./Button.md)`
  - Chromaticや参考URLへのリンクも積極的に追加してください。

## フロントマター (Frontmatter) について
各Markdownファイルの先頭には、`---`で囲まれた「フロントマター」と呼ばれるメタデータが記述されています。

```yaml
---
notion_page_id: "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
category: "Category Name"
---
```

このメタデータは、MarkdownファイルとNotionページを紐付けるための重要な情報です。

- `notion_page_id`: 対応するNotionページのIDです。**このIDは絶対に編集しないでください。**
- `category`: Notionデータベース上のカテゴリです。

**重要:** このフロントマター部分は、Notionページに内容を反映させる際には**除外**されます。同期スクリプトがこの部分を自動的に解釈・処理するため、ページの本文にこの内容が書き込まれることはありません。
