# aegis-lab について

## aegis-lab とは

aegis-lab は、**プロトタイプ量産のための実験場**です。

AI との対話で「言葉」から高精度な画面を生成し、実際に動く UI を爆速で試行錯誤できます。Aegis デザインシステムを使った UI プロトタイプを高速に作成・共有できる開発環境として、アイデアを素早く形にすることができます。

### 対象ユーザー

- **開発者**: React コンポーネントを使ってプロトタイプを実装
- **デザイナー**: デザインの実現可能性を確認、UI の動作検証
- **マネージャー・プロジェクト関係者**: プレビューを通じて画面イメージを共有・レビュー

## なぜ aegis-lab なのか

v0 や bolt.new など、AI を使った UI 生成ツールは他にもあります。それでも aegis-lab を使う理由があります。

### 本物のコンポーネント

他の AI ツールが生成するのは「それっぽい」見た目の UI です。aegis-lab は違います。**製品で実際に使われている Aegis コンポーネントそのもの**を使って画面を作ります。

### 製品の空気感

色、余白、タイポグラフィ、ボタンの挙動——すべてが製品と同じです。プロトタイプの段階から「製品になったらどう見えるか」を正確に確認できます。

### 開発へのショートカット

aegis-lab で作ったコードは、そのまま製品開発に流用できます。プロトタイプから本開発への移行がスムーズになり、作り直しのムダがありません。

## ディレクトリ構成

aegis-lab では、目的に応じてディレクトリが分かれています。

### `/sandbox` - プロトタイプ置き場

あなたが作るプロトタイプはここに格納されます。自由に試行錯誤できる実験場です。

### `/template` - お手本集

既存サービスの画面を再現したテンプレートが用意されています。一覧画面、詳細画面、フォーム画面など、よくあるパターンを参考にできます。

**コツ**: ゼロから作るより、テンプレートを参照して構造を合わせるのが効率的です。

## できること

### UI プロトタイピング

Aegis コンポーネントを使って画面を作成できます。ボタン、フォーム、テーブル、ダイアログなど、製品で使われるコンポーネントをそのまま利用できるため、実際の見た目に近いプロトタイプを作成できます。

### Sandbox ページの作成

`pnpm sandbox:create` コマンドで新しいページを簡単に作成できます。対話形式でページ名やテンプレートを選ぶだけで、すぐに開発を始められます。

### テンプレートの参照

`src/pages/template/` には参考となる実装パターンが用意されています。一覧画面、詳細画面、フォーム画面など、よくあるパターンを参照して効率的に開発できます。

### AI 支援

Claude Code や MCP ツールと連携して、コンポーネントの使い方を調べたり、実装をサポートしてもらうことができます。

### 品質チェック

- **Lint・フォーマット**: コードの品質を自動でチェック
- **日本語テキストチェック**: UI テキストのガイドライン準拠を確認
- **TypeScript 型チェック**: 型エラーを事前に検出

### プレビューデプロイ

PR を作成すると自動でプレビュー環境にデプロイされます。URL を共有するだけで、関係者にプロトタイプを見てもらえます。

プロジェクト単位で URL を固定したい場合は、PR に `preview:{slug}`（例: `preview:clm-report`）ラベルを付けると `https://{slug}-aegis-lab.on-technologies-technical-dept.workers.dev` という固定 URL が発行され、PR をマージしても変わらないスナップショットとして残ります。詳しくは `docs/workflow-guide.md` の「Preview URL Configuration」を参照してください。

## 実際の製品開発環境との違い

aegis-lab は製品開発環境をベースにしていますが、いくつかの点で異なります。

| 観点 | aegis-lab | 製品開発環境 |
|------|-----------|--------------|
| 目的 | 見た目の確認・検証 | 本番運用 |
| コードの複雑さ | シンプル（見た目重視） | 堅牢（エラー処理等完備） |
| Aegis のルール・サンプル | 組み込み済み | 別途参照が必要 |
| 対象ユーザー | エンジニア以外も OK | エンジニア中心 |

非エンジニアでも触れるように、ハードルを下げた設計になっています。
